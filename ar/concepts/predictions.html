التنبؤات وأخذ العيّنات

يُعيد طلب التنبؤ عدّة مستقبلات ممكنة لا واحداً. تشرح هذه الصفحة كيفية قراءة الاستجابة وكيف تصوغها ضوابط أخذ العيّنات. وللاطلاع على المخطط الدقيق راجع مرجع predict.

تشريح الطلب

تتنبأ مقابل ملف بيانات، لا مقابل مصدر بيانات خام. فالملف يعرف سلفاً أي السلاسل يقرأ وكيف يرمّزها، فلا يلزم الطلب إلا أن يذكر أي ملف وكيفية أخذ العيّنات. تغطي ذلك أربعة حقول:

data_profileسلسلة نصيةمطلوب

اسم أحد ملفاتك، مثل hourly_orders — أو ملف عام.

countعدد صحيحمطلوب

كم مساراً مستقبلياً مستقلاً يُسحب. واحد على الأقل.

temperatureرقممطلوب

مدى تشتت العيّنات. يجب أن يكون أكبر من 0؛ ونحو 1.0 محايد.

top_pرقممطلوب

عتبة قطع أخذ العيّنات النووي في النطاق من 0 إلى 1. والأدنى يقصّ الذيل غير المرجّح.

إليك طلباً لعشرين مساراً مقابل ملف hourly_orders:

curl https://api.temporis.co/v1/predict \
  -H "Authorization: Bearer ts_your_token" \
  -H "Content-Type: application/json" \
  -d '{
    "data_profile": "hourly_orders",
    "count": 20,
    "temperature": 1.0,
    "top_p": 0.9
  }'
import requests

resp = requests.post(
    "https://api.temporis.co/v1/predict",
    headers={"Authorization": "Bearer ts_your_token"},
    json={
        "data_profile": "hourly_orders",
        "count": 20,
        "temperature": 1.0,
        "top_p": 0.9,
    },
)
resp.raise_for_status()
result = resp.json()
const resp = await fetch("https://api.temporis.co/v1/predict", {
  method: "POST",
  headers: {
    "Authorization": "Bearer ts_your_token",
    "Content-Type": "application/json",
  },
  body: JSON.stringify({
    data_profile: "hourly_orders",
    count: 20,
    temperature: 1.0,
    top_p: 0.9,
  }),
});
const result = await resp.json();

المرجع الكامل — كل حقل ورمز حالة ورسالة خطأ — موجود في مرجع نقطة نهاية predict.

قراءة الاستجابة

يُعيد النداء الناجح HTTP 200 مع أربعة حقول على المستوى الأعلى:

request_idسلسلة نصية

معرّف سداسي عشري لهذا الطلب. احتفظ به للدعم ولسجلاتك الخاصة.

data_profileسلسلة نصية

يردّد الملف الذي طلبته.

predict_fromعدد صحيح

ثواني يونكس لآخر طابع زمني مرصود يتابع منه التنبؤ. ويبدأ كل مسار بعد هذه النقطة مباشرةً.

predictionsمصفوفة

مدخل واحد لكل مسار مأخوذ كعيّنة (طولها يساوي count). ويحمل كلٌّ منها قيمة loss ومصفوفة data من صفوف التنبؤ.

يعرض المثال أدناه اثنين من المسارات من تنبؤ orders. قيمة interval للملف هي 3600 وpatch_spacing هي 2، فتتقدم الصفوف بمقدار interval × patch_spacing = 7200 ثانية (ساعتان):

{
  "request_id": "9f3c1a7e2b4d",
  "data_profile": "hourly_orders",
  "predict_from": 1718888400,
  "predictions": [
    {
      "loss": 0.84,
      "data": [
        { "timestamp": 1718895600, "orders": 142.0 },
        { "timestamp": 1718902800, "orders": 137.5 }
      ]
    },
    {
      "loss": 1.07,
      "data": [
        { "timestamp": 1718895600, "orders": 151.2 },
        { "timestamp": 1718902800, "orders": 129.8 }
      ]
    }
  ]
}

لكل صف timestamp إضافةً إلى حقل واحد لكل سلسلة يقرؤها الملف — وهنا orders فحسب. وتتقدم الطوابع الزمنية بمقدار interval × patch_spacing ثانية، وهي المباعدة نفسها لكل مسار، فتستطيع محاذاة العيّنات صفاً بصف.

ضوابط أخذ العيّنات

تصوغ ثلاثة معاملات مجموعة المستقبلات التي تستعيدها. وهي تقايض الدقة والتنوع مقابل الكلفة والاستقرار.

المعاملما الذي يفعلهالقيمة النموذجيةارفعه / اخفضه عندما
countعدد المسارات المستقلة المسحوبة. عيّنات أكثر تحلّ التوزيع على نحو أفضل.20 إلى 100ارفعه لفترات أكثر نعومة؛ اخفضه لتقليص كلفة الرموز.
temperatureمدى تشتت العيّنات. الأعلى أكثر تنوعاً وأكثر عدم يقين؛ والأدنى يتجمّع قرب المسار الأرجح.~1.0ارفعه عندما تبدو التنبؤات ضيقة جداً؛ اخفضه عندما تتبعثر على نحو غير واقعي.
top_pعتبة القطع النووي. تقصّ الذيل غير المرجّح قبل أخذ العيّنات.0.9 إلى 1.0اخفضه لكبح القفزات المفاجئة؛ ارفعه نحو 1.0 للإبقاء على التوزيع الكامل.
الكلفة تتناسب مع count

تُولَّد كل عيّنة على نحو مستقل وتُحسب بحسب كل رمز من استخدام النموذج، فطلب بـcount 100 يكلّف نحو خمسة أمثال طلب بـcount 20. اختر count لتوازن الدقة التي تحتاجها مقابل الكلفة.

الخسارة: ترتيب المسارات

يحمل كل تنبؤ قيمة loss. الأدنى يعني أن النموذج عدّ ذلك المسار أكثر نمطية — أكثر اتساقاً مع ما توقّعه بالنظر إلى التاريخ. والمسارات ذات الخسارة الأعلى تبقى عيّنات صالحة؛ وهي فقط أقل مركزية.

استخدم loss لانتقاء مسار ممثِّل واحد، أو كوزن عند تلخيص المجموعة. ولالتقاط المسار الأكثر نمطية:

best = min(result["predictions"], key=lambda p: p["loss"])

التنبؤات متعدّدة المتغيّرات

إذا قرأ ملف عدّة أسماء سلاسل، فإن كل صف تنبؤ يحمل حقلاً واحداً لكل اسم — تُتنبّأ السلاسل مجتمعةً، فتبقى قيمها متسقة داخل الصف. وملف يقرأ kw وprice معاً يُعيد صفوفاً مثل:

{ "timestamp": 1718895600, "kw": 482.0, "price": 31.7 }

ملفات البيانات العامة

تستطيع التنبؤ مقابل ملفات مشتركة يصونها Temporis، دون أي إعداد من جانبك — مرّر فقط الاسم العام كـdata_profile:

{ "data_profile": "temporis/polymarket", "count": 20, "temperature": 1.0, "top_p": 0.9 }

شكل الاستجابة متطابق؛ وحقول السلاسل في كل صف تطابق ما يقرؤه الملف العام.
