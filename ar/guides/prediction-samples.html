العمل مع عينات التنبؤ

يُرجِع /v1/predict مسارات معيَّنة كثيرة. يحوّلها هذا الدليل إلى الأشياء التي تحتاجها فعلًا: تنبؤ نقطي ونطاق ثقة.

لماذا مسارات كثيرة

يولّد Temporis التنبؤات عبر التعيين — فهو يقرأ سجلك التاريخي ويرسم امتدادًا معقولًا، تمامًا كما يكمل نموذج لغوي نصًا. كل سحبة هي مستقبل واحد ممكن، لا المستقبل بعينه. اطلب عدد count سحبات وستحصل على عدد count من المسارات المستقلة في predictions.

الغاية من امتلاك العديد منها هي التشتت. حيث تتفق المسارات يكون النموذج واثقًا؛ وحيث تتباعد يكون المستقبل غير مؤكد فعلًا. ذلك التشتت هو تقديرك لعدم اليقين — ولهذا نادرًا ما تكون عينة واحدة هي ما تريده.

اضبط count عاليًا بما يكفي لتمييز ذلك التشتت. حفنة من السحبات تعطي صورة مهتزة وغير موثوقة؛ بينما نطاق يتراوح بين 20 و100 يميّز التوزيع جيدًا لأغلب السلاسل. المزيد أكثر سلاسة لكنه ليس مجانيًا.

العدد يكلّف رموزًا

يُحتسب التنبؤ مقابل كل رمز نموذج، وكل مسار يولّد أفقًا كاملًا من الرموز. مضاعفة count تضاعف تقريبًا تكلفة الطلب. استخدم عينات كافية لرؤية التشتت بوضوح، ثم توقّف.

تنبؤ نقطي

لتقليص العينات إلى رقم واحد لكل طابع زمني مستقبلي، جمّع عبر المسارات عند كل خطوة زمنية. الوسيط خيار افتراضي جيد — فهو متين أمام المسار الشاذ العَرَضي؛ والمتوسط يفي بالغرض أيضًا إن فضّلته.

تتشارك صفوف data في كل تنبؤ الطوابع الزمنية نفسها بالترتيب نفسه، فيمكنك محاذاتها بالفهرس. هنا نطلب 50 عينة من ملف hourly_orders ونحسب تنبؤ orders بالوسيط لكل طابع زمني:

import os, requests
from statistics import median

token = os.environ["TEMPORIS_TOKEN"]

resp = requests.post(
    "https://api.temporis.co/v1/predict",
    headers={"Authorization": f"Bearer {token}"},
    json={"data_profile": "hourly_orders", "count": 50, "temperature": 1.0, "top_p": 0.9},
)
resp.raise_for_status()
body = resp.json()

# Each prediction's rows are in the same timestamp order, so transpose by index.
preds = body["predictions"]            # length == count (50)
rows = list(zip(*[p["data"] for p in preds]))  # group row i across all samples

point_forecast = []
for step in rows:
    ts = step[0]["timestamp"]          # shared timestamp for this step
    values = [row["orders"] for row in step]
    point_forecast.append({"timestamp": ts, "orders": median(values)})

for f in point_forecast[:3]:
    print(f["timestamp"], round(f["orders"], 1))

نطاقات الثقة

الوسيط يعطيك خطًا مركزيًا؛ والنطاق يعطيك المدى حوله. احسب الكميات لكل طابع زمني عبر العينات — المئينات العاشر والخمسون والتسعون خيار شائع. والنتيجة ثلاثة خطوط: حافة دنيا (p10)، والمركز (p50)، وحافة عليا (p90).

اقرأ النطاق على أنه "نحو 80% من المستقبلات المعيَّنة تقع بين p10 و p90 عند هذا الطابع الزمني." النطاق الضيق يعني أن النموذج واثق؛ والنطاق الذي يتسع كلما نظرت أبعد إلى الأمام هو النموذج وهو يخبرك بصدق أن المستقبل البعيد أقل تأكيدًا من المستقبل القريب.

import os, requests
import numpy as np

token = os.environ["TEMPORIS_TOKEN"]

resp = requests.post(
    "https://api.temporis.co/v1/predict",
    headers={"Authorization": f"Bearer {token}"},
    json={"data_profile": "hourly_orders", "count": 50, "temperature": 1.0, "top_p": 0.9},
)
resp.raise_for_status()
body = resp.json()

preds = body["predictions"]
timestamps = [row["timestamp"] for row in preds[0]["data"]]

# Shape: (count, horizon) — one row per sample, one column per timestep.
matrix = np.array([[row["orders"] for row in p["data"]] for p in preds])

p10, p50, p90 = np.percentile(matrix, [10, 50, 90], axis=0)

band = [
    {"timestamp": ts, "p10": lo, "p50": mid, "p90": hi}
    for ts, lo, mid, hi in zip(timestamps, p10, p50, p90)
]
for b in band[:3]:
    print(b["timestamp"], round(b["p10"], 1), round(b["p50"], 1), round(b["p90"], 1))

اختيار count و temperature و top_p عمليًا

ثلاثة مقابض تشكّل العينات، ولكل منها وظيفة مختلفة:

  • count يتحكم في الدقة — كم بدقة يُعيَّن التشتت. لا يغيّر شكل التوزيع، بل وضوح رؤيتك له فقط.
  • temperature (أكبر من 0) يتحكم في التشتت — القيم الأعلى تسحب مستقبلات أكثر تنوعًا وأوسع مدى؛ والقيم الأدنى تجمّع العينات قرب المسار الأرجح للنموذج. نحو 1.0 محايد.
  • top_p (من 0 إلى 1) يقصّ الذيول — خفضه يقطع القيم القصوى غير المرجّحة، فتحصل على قفزات مفاجئة أقل؛ والقيمة 1.0 تُبقي التوزيع كاملًا.

نقاط بداية معقولة، تبعًا لمدى تقلّب سلسلتك:

طابع السلسلةcounttemperaturetop_p
مستقرة، سلسة (طلب ثابت)20–300.80.9
نموذجية501.00.9
متقلبة، حادة (الأسعار، حركة مرور متفجّرة)80–1001.10.95

عامِل هذه كنقطة انطلاق. ارسم نطاقًا، وانظر هل هو ضيق جدًا بحيث لا يكون صادقًا أم واسع جدًا بحيث يكون عديم الفائدة، وعدّل temperature أولًا.

الترتيب حسب الخسارة

أحيانًا تريد مسارًا تمثيليًا واحدًا بدلًا من نطاق — لعرض توضيحي، أو فحص سلامة، أو سؤال سريع "كيف يبدو مستقبل نموذجي". كل مسار يحمل قيمة loss، وخسارة أقل تعني أن النموذج اعتبر ذلك المسار أكثر نموذجية. رتّب حسبها وخذ مقدمة القائمة:

# body is the parsed /v1/predict response
ranked = sorted(body["predictions"], key=lambda p: p["loss"])

most_typical = ranked[0]               # lowest loss = most representative path
for row in most_typical["data"][:3]:
    print(row["timestamp"], row["orders"])

# Or trim outliers: keep the most-typical 80% before computing a band.
keep = ranked[: int(len(ranked) * 0.8)]

لتنبؤ فعلي، فضّل الوسيط ونطاقًا على أي مسار مفرد — فالمسار الأدنى خسارةً مستقبل واحد معقول، لا متوسط لها. الترتيب أكثر فائدة لانتقاء مثال نظيف أو تصفية القيم الشاذة.
