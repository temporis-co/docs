Authentifizierung

Jede Temporis-API-Anfrage authentifiziert sich mit einem Bearer-Token. Hier erfahren Sie, wie Sie eines erstellen, senden und sicher aufbewahren.

Zugriffstoken

Sie erstellen Zugriffstoken im Dashboard unter Access Tokens. Ein Token ist eine lange zufällige Zeichenfolge mit dem Präfix ts_.

Das vollständige Token wird einmalig angezeigt, im Moment seiner Erstellung. Temporis speichert serverseitig nur einen Hash davon, niemals das Token selbst. Das bedeutet, dass ein verlorenes Token nicht wiederhergestellt werden kann — wenn Sie es verlieren, löschen Sie es und erstellen Sie ein neues.

Behandeln Sie Token wie Passwörter

Jeder, der eines Ihrer Token besitzt, kann Daten aufnehmen und Prognosen ausführen — und Prognosen werden Ihrem Konto in Rechnung gestellt. Schützen Sie sie entsprechend.

Authentifizierung einer Anfrage

Senden Sie das Token bei jeder Anfrage im Authorization-Header, unter Verwendung des Bearer-Schemas:

Authorization: Bearer ts_your_token_here

Die Beispiele in dieser Dokumentation lesen das Token aus einer Umgebungsvariablen TEMPORIS_TOKEN, sodass das Geheimnis niemals im Quellcode erscheint. Exportieren Sie es einmal in Ihrer Shell:

export TEMPORIS_TOKEN="ts_your_token_here"

Hängen Sie es dann als Header an:

curl https://api.temporis.co/v1/predict \
  -H "Authorization: Bearer $TEMPORIS_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{ "data_profile": "hourly_orders", "count": 5, "temperature": 1.0, "top_p": 0.9 }'
import os, requests

token = os.environ["TEMPORIS_TOKEN"]

resp = requests.post(
    "https://api.temporis.co/v1/predict",
    headers={"Authorization": f"Bearer {token}"},
    json={"data_profile": "hourly_orders", "count": 5, "temperature": 1.0, "top_p": 0.9},
)
resp.raise_for_status()
const token = process.env.TEMPORIS_TOKEN;

const resp = await fetch("https://api.temporis.co/v1/predict", {
  method: "POST",
  headers: {
    "Authorization": `Bearer ${token}`,
    "Content-Type": "application/json",
  },
  body: JSON.stringify({ data_profile: "hourly_orders", count: 5, temperature: 1.0, top_p: 0.9 }),
});

Wenn die Authentifizierung fehlschlägt

Wenn der Authorization-Header fehlt oder fehlerhaft ist oder das Token unbekannt ist, antwortet die API mit HTTP 401 und einem JSON-Body, der beschreibt, was schiefgelaufen ist:

{ "detail": "Invalid access token." }

Ein fehlender oder fehlerhafter Header gibt stattdessen { "detail": "Missing access token." } zurück. Beides sind 401er. In Fehler & Statuscodes finden Sie die vollständige Liste der Antworten.

Token sicher aufbewahren

Ein paar Gewohnheiten halten Ihre Token von unbefugten Händen fern:

  • Committen Sie Token niemals in die Versionsverwaltung und platzieren Sie sie niemals in clientseitigem Code. Browser und mobile Apps liefern ihren Quellcode an die Benutzer aus.
  • Lesen Sie Token aus Umgebungsvariablen oder einem Secrets-Manager. Halten Sie sie aus Logs, Screenshots und Tickets heraus.
  • Erstellen Sie ein separates Token pro Dienst, damit Sie eines widerrufen können, ohne die anderen zu stören.
  • Löschen Sie ein Token im Dashboard, um es sofort zu widerrufen. Die Löschung wird sofort wirksam — die nächste Anfrage, die dieses Token verwendet, erhält ein 401.
