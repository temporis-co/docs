Datensätze aufnehmen

Senden Sie Beobachtungen in eine Datenquelle. Die Datenquelle muss bereits existieren (erstellen Sie sie im Dashboard) und darf nicht archiviert sein.

POST/v1/data_sources/ingest

Request-Body

Der Body benennt die Ziel-Datenquelle und transportiert einen Stapel von Datensätzen.

data_sourcestringErforderlich

Der Name einer existierenden, nicht archivierten Datenquelle, in die geschrieben werden soll.

recordsarrayErforderlich

Eine Liste von Datensätzen, die gespeichert werden sollen. Jeder Datensatz ist eine einzelne Beobachtung einer Reihe zu einem Zeitpunkt.

Datensatzfelder

timestampintegerErforderlich

Unix-Zeit in Sekunden.

namestringErforderlich

Der Name der Reihe. Eine Datenquelle enthält viele benannte Reihen.

valuenumberErforderlich

Der beobachtete Wert (eine Ganzzahl oder ein Gleitkommawert).

Antwort

Bei Erfolg gibt der Endpunkt 204 No Content mit einem leeren Body zurück:

204 No Content

Upsert-Verhalten

Jeder Datensatz wird durch das Paar (timestamp, name) eindeutig identifiziert. Das erneute Senden eines Datensatzes mit demselben Zeitstempel und Reihennamen überschreibt den gespeicherten Wert.

Wiederholungen und Backfills sind idempotent

Da der Schreibvorgang ein Upsert ist, können Sie eine fehlgeschlagene Anfrage gefahrlos wiederholen oder historische Daten erneut senden — Duplikate fallen auf denselben Schlüssel zusammen, anstatt sich anzuhäufen.

Batching

Senden Sie bevorzugt viele Datensätze in einer einzigen Anfrage statt einen Datensatz pro Anfrage. Batching reduziert Round Trips und Overhead drastisch. Wenn Sie Historie laden, senden Sie sie in Blöcken von jeweils vielen Datensätzen, anstatt einen Punkt nach dem anderen zu streamen.

Fehler

Fehler geben den HTTP-Status sowie einen JSON-Body der Form { "detail": "..." } zurück.

Statusdetail-MeldungBedeutung
401Invalid access token. / Missing access token.Authentifizierung fehlgeschlagen — überprüfen Sie Ihr Token.
404Data source not found.Die genannte Datenquelle existiert nicht. Erstellen Sie sie zuerst im Dashboard.
409Data source is archived.Die Datenquelle ist archiviert und kann keine Schreibvorgänge annehmen. Heben Sie die Archivierung auf oder verwenden Sie eine andere Quelle.

In Fehler & Statuscodes finden Sie die vollständige Referenz.
