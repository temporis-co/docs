Prognose erstellen

Ziehen Sie eine oder mehrere gesampelte Zukünfte aus einem Datenprofil. Erfordert eine hinterlegte Karte.

POST/v1/predict

Anfrage-Body

Sie benennen ein Datenprofil und die gewünschte Form des Samples.

data_profilestringErforderlich

Der Name eines Ihrer Profile oder eines öffentlichen Profils wie temporis/polymarket.

countintegerErforderlich

Die Anzahl der unabhängigen Zukunfts-Trajektorien, die gezogen werden sollen. Muss mindestens 1 sein.

temperaturenumberErforderlich

Sampling-Temperatur, größer als 0. Höhere Werte streuen die Samples weiter; niedrigere halten sie nahe am wahrscheinlichsten Pfad.

top_pnumberErforderlich

Nucleus-Sampling-Grenzwert im Bereich von 0 bis 1. Niedrigere Werte schneiden den unwahrscheinlichen Rand ab; 1.0 behält die vollständige Verteilung bei.

count, temperature und top_p werden ausführlich unter Prognosen erklärt.

Beispielanfrage

Diese zieht fünf Trajektorien aus dem Profil hourly_orders bei neutraler Temperatur:

curl https://api.temporis.co/v1/predict \
  -H "Authorization: Bearer $TEMPORIS_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "data_profile": "hourly_orders",
    "count": 5,
    "temperature": 1.0,
    "top_p": 0.9
  }'

Antwortfelder

Eine erfolgreiche Anfrage gibt HTTP 200 mit diesem Body zurück.

request_idstring

Ein eindeutiger Bezeichner für diese Prognoseanfrage.

data_profilestring

Gibt den von Ihnen angefragten Profilnamen zurück.

predict_frominteger

Unix-Sekunden: der letzte beobachtete Zeitstempel, ab dem die Prognose fortgesetzt wird. Jede Trajektorie schreitet von hier aus voran.

predictionsarray

Die gesampelten Trajektorien. Die Länge entspricht dem von Ihnen angefragten count.

Prediction-Objekt

lossnumber

Niedriger bedeutet, dass das Modell diese Trajektorie als typischer einstuft. Verwenden Sie ihn, um Samples zu ordnen oder ein repräsentatives Sample auszuwählen.

dataarray

Die Prognosezeilen. Jede Zeile hat einen timestamp plus ein Feld pro Reihenname im Profil.

Öffentliche Datenprofile

Sie können einen geteilten Datensatz ganz ohne eigene Einrichtung prognostizieren, indem Sie einen öffentlichen Profilnamen als data_profile übergeben. Diese werden von Temporis gepflegt und sind bereit zur Auslieferung:

{ "data_profile": "temporis/polymarket", "count": 3, "temperature": 1.0, "top_p": 0.9 }

Fehler

Fehler geben den HTTP-Status plus einen JSON-Body der Form { "detail": "..." } zurück.

Statusdetail-MeldungBedeutung
400Invalid prediction parameters.Ein Parameter liegt außerhalb des zulässigen Bereichs, etwa count unter 1 oder temperature nicht größer als 0.
402Add a credit card before using the API. / There is a payment issue. Add a new credit card.Keine aktive Abrechnung. Fügen Sie im Dashboard eine Karte hinzu oder korrigieren Sie sie.
404Data profile not found.Es existiert kein Profil mit diesem Namen.
404Data profile is not ready to serve.Das Profil existiert, hat aber die Vorbereitung noch nicht abgeschlossen.
409Not enough data for this data profile.Das Profil verfügt über weniger als seine mindestens erforderliche Historie. Nehmen Sie mehr Daten auf und versuchen Sie es erneut.
502Prediction failed.Ein vorübergehender serverseitiger Fehler. Ein erneuter Versuch mit Backoff ist sicher.
