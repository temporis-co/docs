Datenprofile

Ein Datenprofil ist das Rezept, das Temporis vorgibt, wie eine Datenquelle gelesen wird: welche Reihen, in welcher Taktung, wie viel Historie und wie sie bereinigt wird. Es ist das, wogegen Sie Prognosen erstellen.

Was ein Profil definiert

Eine Datenquelle enthält Rohdatenzeilen. Ein Profil verwandelt diese Zeilen in ein sauberes, regelmäßig getaktetes Signal, das das Modell lesen kann — es wählt die Reihen aus, ordnet Beobachtungen einem Zeitraster zu, entscheidet, wie viel Historie verwendet wird, und bereinigt Lücken. Wenn Sie predict aufrufen, benennen Sie ein Profil, nicht eine Quelle.

Profile erstellen und bearbeiten Sie im Dashboard. Das Beispiel hier ist hourly_orders: Es liest die Reihe orders aus store_metrics auf einem Ein-Stunden-Raster (interval 3600).

Auswahl der Daten

Quelle und Reihen

Zwei Einstellungen bestimmen, was das Profil liest:

  • data_source — aus welcher Datenquelle gelesen wird.
  • names — eine oder mehrere Reihennamen innerhalb dieser Quelle.

Die Angabe eines einzelnen Namens ergibt eine univariate Prognose. Die Angabe mehrerer — etwa orders und visitors — macht das Profil multivariat: Die Reihen werden gemeinsam prognostiziert, sodass das Modell die Beziehung zwischen ihnen nutzen kann.

Das Zeitraster

Echte Beobachtungen treffen selten in einer perfekt gleichmäßigen Taktung ein, daher resampelt ein Profil sie auf ein regelmäßiges Raster.

interval ist die Größe jedes Rasterbuckets in Sekunden. Mit interval 3600 werden Beobachtungen in Stundenbuckets eingeordnet. Das Raster wird an Ihrer jüngsten Beobachtung verankert und schreitet von dort rückwärts, sodass die aktuellsten Daten immer sauber auf dem Raster landen.

Das Modell liest nicht jeden Punkt einzeln. Es arbeitet in Patches — Abschnitten der Reihe mit fester Größe:

  • patch_len — aus wie vielen Punkten ein Patch besteht.
  • patch_spacing — der Schritt zwischen den verwendeten Punkten, eine Form des Downsamplings.
  • num_patches — die meisten Patches an Historie, die als Kontext verwendet werden.
  • min_patches — die wenigsten Patches an Historie, die erforderlich sind, um eine Prognose zu liefern.

Anschaulich: Der Kontext, den das Modell sieht, sind ungefähr num_patches Patches aktueller Historie. Wenn ein Profil weniger als min_patches Patches verfügbar hat, kann es nicht liefern, und Sie erhalten HTTP 409 "Not enough data for this data profile."

Das Spacing legt auch Ihre Prognoseauflösung fest. Der effektive Schritt zwischen Prognosepunkten beträgt interval × patch_spacing Sekunden. Mit interval 3600 und patch_spacing 1 liegen die Prognosen eine Stunde auseinander.

Bereinigung der Reihe

Nach dem Resampling kann es Buckets ohne Beobachtung geben. Zwei Einstellungen entscheiden, wie das gehandhabt wird:

  • fill_limit — die längste Folge aufeinanderfolgender leerer Buckets, die per Forward-Fill aufgefüllt wird (der letzte bekannte Wert wird fortgeführt). Eine längere Lücke wird als tatsächlich fehlend behandelt, und diese Zeilen werden stattdessen verworfen.
  • exclude_window — ein wiederkehrendes Tageszeitfenster, das ausgeblendet wird, angegeben als Zeitzone plus Start und Ende. Verwenden Sie es für erwartete Totzeiten, etwa Nachtstunden, in denen ein Markt geschlossen ist, damit diese ruhige Phase die Reihe nicht verzerrt.

Niveaus vs. Wachstum

Manche Reihen lassen sich am besten über ihr Niveau modellieren, andere über ihre relative Veränderung. Die Einstellung use_diffs (die Log-Diff-Transformation) schaltet das Modell darauf um, mit logarithmischem Wachstum zu arbeiten — proportionale Veränderung von einem Schritt zum nächsten — statt mit dem Rohwert.

Aktivieren Sie sie für Reihen, die sich multiplikativ bewegen oder über einen weiten Bereich stark trenden: Umsatz, Preise, alles, bei dem eine Bewegung um 10 % wichtiger ist als eine absolute. Lassen Sie sie deaktiviert für Reihen, die in einem begrenzten, ungefähr stationären Bereich liegen, wo das Rohniveau bereits das natürliche Vorhersageziel ist.

Bereitschaft

Ein Profil ist nicht in dem Moment nutzbar, in dem Sie es erstellen. Temporis berechnet zunächst Normalisierungsstatistiken — es lernt die typische Streuung Ihrer Daten, um Werte in den vom Modell erwarteten Bereich zu skalieren. Bis genügend Daten dafür vorliegen, ist das Profil nicht bereit, und das Dashboard zeigt seinen Bereitschaftsstatus an.

Eine Prognose gegen ein Profil, das nicht bereit ist, gibt HTTP 404 "Data profile is not ready to serve." zurück. Warten Sie, bis das Dashboard das Profil als bereit meldet, und rufen Sie dann predict auf.
