Mit Prognose-Samples arbeiten

/v1/predict gibt viele gesampelte Trajektorien zurück. Diese Anleitung verwandelt sie in das, was Sie tatsächlich brauchen: eine Punktprognose und ein Konfidenzband.

Warum viele Trajektorien

Temporis erzeugt Prognosen durch Sampling — es liest Ihre Historie und zeichnet eine plausible Fortsetzung, so wie ein Sprachmodell Text fortsetzt. Jede Ziehung ist eine mögliche Zukunft, nicht die Zukunft. Fordern Sie count Ziehungen an, erhalten Sie count unabhängige Trajektorien in predictions.

Der Sinn vieler Ziehungen ist die Streuung. Wo die Trajektorien übereinstimmen, ist das Modell zuversichtlich; wo sie auseinanderlaufen, ist die Zukunft tatsächlich ungewiss. Diese Streuung ist Ihre Unsicherheitsschätzung — weshalb ein einzelnes Sample selten das ist, was Sie wollen.

Setzen Sie count hoch genug, um diese Streuung aufzulösen. Eine Handvoll Ziehungen gibt ein zappeliges, unzuverlässiges Bild; irgendwo im Bereich von 20 bis 100 löst die Verteilung für die meisten Reihen gut auf. Mehr ist glatter, aber nicht kostenlos.

Count kostet Tokens

Prognosen werden pro Modell-Token abgerechnet, und jede Trajektorie erzeugt einen vollen Horizont an Tokens. Eine Verdopplung von count verdoppelt ungefähr die Kosten des Aufrufs. Verwenden Sie genug Samples, um die Streuung klar zu sehen, und hören Sie dann auf.

Eine Punktprognose

Um die Samples in eine Zahl pro zukünftigem Zeitstempel zusammenzufassen, aggregieren Sie über die Trajektorien hinweg bei jedem Zeitschritt. Der Median ist ein guter Standard — er ist robust gegenüber der gelegentlichen Ausreißer-Trajektorie; der Mittelwert funktioniert ebenfalls, wenn Sie ihn bevorzugen.

Die data-Zeilen jeder Prognose teilen dieselben Zeitstempel in derselben Reihenfolge, sodass Sie sie nach Index ausrichten können. Hier fordern wir 50 Samples vom Profil hourly_orders an und berechnen eine mediane orders-Prognose pro Zeitstempel:

import os, requests
from statistics import median

token = os.environ["TEMPORIS_TOKEN"]

resp = requests.post(
    "https://api.temporis.co/v1/predict",
    headers={"Authorization": f"Bearer {token}"},
    json={"data_profile": "hourly_orders", "count": 50, "temperature": 1.0, "top_p": 0.9},
)
resp.raise_for_status()
body = resp.json()

# Each prediction's rows are in the same timestamp order, so transpose by index.
preds = body["predictions"]            # length == count (50)
rows = list(zip(*[p["data"] for p in preds]))  # group row i across all samples

point_forecast = []
for step in rows:
    ts = step[0]["timestamp"]          # shared timestamp for this step
    values = [row["orders"] for row in step]
    point_forecast.append({"timestamp": ts, "orders": median(values)})

for f in point_forecast[:3]:
    print(f["timestamp"], round(f["orders"], 1))

Konfidenzbänder

Der Median gibt Ihnen eine Mittellinie; ein Band gibt Ihnen die Spanne darum herum. Berechnen Sie Quantile pro Zeitstempel über die Samples hinweg — das 10., 50. und 90. Perzentil sind eine gängige Wahl. Das Ergebnis sind drei Linien: eine untere Kante (p10), die Mitte (p50) und eine obere Kante (p90).

Lesen Sie das Band als "etwa 80 % der gesampelten Zukünfte fallen zu diesem Zeitstempel zwischen p10 und p90". Ein schmales Band bedeutet, das Modell ist zuversichtlich; ein Band, das sich weitet, je weiter Sie nach vorne blicken, ist das Modell, das Ihnen ehrlich sagt, dass die ferne Zukunft weniger sicher ist als die nahe.

import os, requests
import numpy as np

token = os.environ["TEMPORIS_TOKEN"]

resp = requests.post(
    "https://api.temporis.co/v1/predict",
    headers={"Authorization": f"Bearer {token}"},
    json={"data_profile": "hourly_orders", "count": 50, "temperature": 1.0, "top_p": 0.9},
)
resp.raise_for_status()
body = resp.json()

preds = body["predictions"]
timestamps = [row["timestamp"] for row in preds[0]["data"]]

# Shape: (count, horizon) — one row per sample, one column per timestep.
matrix = np.array([[row["orders"] for row in p["data"]] for p in preds])

p10, p50, p90 = np.percentile(matrix, [10, 50, 90], axis=0)

band = [
    {"timestamp": ts, "p10": lo, "p50": mid, "p90": hi}
    for ts, lo, mid, hi in zip(timestamps, p10, p50, p90)
]
for b in band[:3]:
    print(b["timestamp"], round(b["p10"], 1), round(b["p50"], 1), round(b["p90"], 1))

count, temperature, top_p in der Praxis wählen

Drei Stellschrauben formen die Samples, und sie erfüllen unterschiedliche Aufgaben:

  • count steuert die Auflösung — wie fein die Streuung gesampelt wird. Es ändert nicht die Form der Verteilung, nur wie klar Sie sie sehen.
  • temperature (größer als 0) steuert die Streuung — höhere Werte ziehen vielfältigere, weiter gestreute Zukünfte; niedrigere Werte gruppieren die Samples nahe am wahrscheinlichsten Pfad des Modells. Etwa 1.0 ist neutral.
  • top_p (0 bis 1) beschneidet die Ränder — eine Senkung schneidet die unwahrscheinlichen Extreme ab, sodass Sie weniger überraschende Sprünge erhalten; 1.0 behält die volle Verteilung.

Sinnvolle Ausgangspunkte, abhängig davon, wie sprunghaft Ihre Reihe ist:

Charakter der Reihecounttemperaturetop_p
Stabil, glatt (gleichmäßige Nachfrage)20–300.80.9
Typisch501.00.9
Volatil, sprunghaft (Preise, stoßartiger Traffic)80–1001.10.95

Behandeln Sie diese als Ausgangspunkt. Zeichnen Sie ein Band, schauen Sie, ob es zu eng ist, um ehrlich zu sein, oder so weit, dass es nutzlos ist, und passen Sie zuerst temperature an.

Nach Loss sortieren

Manchmal möchten Sie einen einzelnen repräsentativen Pfad statt eines Bandes — für eine Demo, einen Plausibilitätscheck oder ein schnelles "wie sieht eine typische Zukunft aus". Jede Trajektorie trägt einen loss, und ein niedrigerer Loss bedeutet, dass das Modell diesen Pfad für typischer hielt. Sortieren Sie danach und nehmen Sie den Anfang der Liste:

# body is the parsed /v1/predict response
ranked = sorted(body["predictions"], key=lambda p: p["loss"])

most_typical = ranked[0]               # lowest loss = most representative path
for row in most_typical["data"][:3]:
    print(row["timestamp"], row["orders"])

# Or trim outliers: keep the most-typical 80% before computing a band.
keep = ranked[: int(len(ranked) * 0.8)]

Für eine tatsächliche Prognose bevorzugen Sie den Median und ein Band gegenüber jeder einzelnen Trajektorie — der Pfad mit dem niedrigsten Loss ist eine plausible Zukunft, nicht ein Durchschnitt davon. Das Sortieren ist am nützlichsten, um ein sauberes Beispiel auszuwählen oder Ausreißer herauszufiltern.
