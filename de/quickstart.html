Schnellstart

Diese Anleitung führt Sie von einem frischen Konto zu Ihrer ersten Prognose. Sie erstellen einen API-Schlüssel, registrieren eine Datenquelle, streamen etwas Historie hinein, bauen ein Datenprofil und rufen den Prognose-Endpunkt auf. Planen Sie etwa fünf Minuten ein.

Zwei Oberflächen, eine Plattform. Sie richten die Dinge im Dashboard ein (Schlüssel, Datenquellen, Datenprofile, Abrechnung) und erledigen die Daten- und Prognosearbeit über die API. Diese Seite führt durch beides.

  1. Ein Zugriffstoken erstellen

    Öffnen Sie im Dashboard Access Tokens und erstellen Sie eines. Das Token wird einmal angezeigt — kopieren Sie es sofort. Es sieht aus wie ts_… und authentifiziert jede API-Anfrage.

    Speichern Sie es als Umgebungsvariable, damit die folgenden Beispiele unverändert funktionieren:

    export TEMPORIS_TOKEN="ts_your_token_here"

  2. Eine Zahlungsmethode hinzufügen

    Die Prognose wird pro genutztem Token abgerechnet, daher ist eine hinterlegte Karte erforderlich, bevor /v1/predict ausgeführt wird. Fügen Sie eine unter Billing hinzu. Das Aufnehmen von Daten erfordert keine Abrechnung — nur die Prognose.

  3. Eine Datenquelle erstellen

    Erstellen Sie unter Data Sources eine mit dem Namen store_metrics. Namen müssen mit einem Kleinbuchstaben beginnen und dürfen nur Kleinbuchstaben, Ziffern und Unterstriche enthalten. Eine Datenquelle ist ein leerer Container, bis Sie ihr Daten senden.

  4. Etwas Historie senden

    Jeder Datensatz ist ein (timestamp, name, value)-Tripel. Der timestamp ist eine Unix-Zeit in Sekunden, name ist die Reihe und value ist die Zahl. Hier senden wir eine Handvoll stündlicher orders-Messwerte; in der Praxis würden Sie so viel echte Historie senden, wie Sie haben.

    Eine erfolgreiche Aufnahme gibt 204 No Content zurück — einen leeren Body. Wird dasselbe (timestamp, name) erneut gesendet, wird der Wert einfach überschrieben, sodass Wiederholungen und Nachträge unbedenklich sind. Details finden Sie unter Datensätze aufnehmen.

  5. Ein Datenprofil aufbauen

    Öffnen Sie zurück im Dashboard Data Profiles und erstellen Sie eines namens hourly_orders. Richten Sie es auf die Datenquelle store_metrics, wählen Sie die Reihe orders und setzen Sie das Intervall auf 3600 Sekunden (eine Stunde). Die Standardwerte für alles Übrige sind ein guter Ausgangspunkt.

    Ein Profil braucht eine Grundmenge an Historie, bevor Temporis es für die Auslieferung vorbereiten kann. Sobald es genügend Daten hat, markiert das Dashboard es als ready. Bis dahin geben Prognosen einen „not ready"-Fehler zurück — das ist zu erwarten.

  6. Eine Prognose erzeugen

    Fragen Sie nun das Profil nach dem, was als Nächstes kommt. count ist die Anzahl unabhängiger Zukunftstrajektorien, die gezogen werden; temperature und top_p steuern, wie risikofreudig das Sampling ist. 

    curl -X POST "https://api.temporis.co/v1/predict" \
  -H "Authorization: Bearer $TEMPORIS_TOKEN" \
  --json '{
    "data_profile": "hourly_orders",
    "count": 5,
    "temperature": 1.0,
    "top_p": 0.9
  }'

Was gerade passiert ist

Sie haben Beobachtungen in eine Datenquelle gestreamt, mit einem Datenprofil beschrieben, wie sie zu lesen sind, und fünf mögliche Zukünfte aus /v1/predict gezogen. Jede Prognose ist eine vollständige Trajektorie; zusammen bilden sie ein probabilistisches Bild davon, was als Nächstes kommt. Je niedriger der loss einer Prognose, desto typischer hielt das Modell diese Trajektorie.
