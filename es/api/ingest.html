Ingerir registros

Envía observaciones a una fuente de datos. La fuente de datos ya debe existir (créala en el panel) y no estar archivada.

POST/v1/data_sources/ingest

Cuerpo de la solicitud

El cuerpo indica la fuente de datos de destino y transporta un lote de registros.

data_sourcestringObligatorio

El nombre de una fuente de datos existente y no archivada en la que escribir.

recordsarrayObligatorio

Una lista de registros que almacenar. Cada registro es una única observación de una serie en un momento dado.

Campos del registro

timestampintegerObligatorio

Tiempo Unix en segundos.

namestringObligatorio

El nombre de la serie. Una fuente de datos contiene muchas series con nombre.

valuenumberObligatorio

El valor observado (un entero o un decimal).

Solicitud de ejemplo

Esto escribe cuatro observaciones de orders en la fuente de datos store_metrics, leyendo el token del entorno:

curl https://api.temporis.co/v1/data_sources/ingest \
  -H "Authorization: Bearer $TEMPORIS_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "data_source": "store_metrics",
    "records": [
      { "timestamp": 1718841600, "name": "orders", "value": 42 },

      { "timestamp": 1718848800, "name": "orders", "value": 47 },
      { "timestamp": 1718852400, "name": "orders", "value": 63 }
    ]
  }'

Respuesta

Si tiene éxito, el endpoint devuelve 204 No Content con un cuerpo vacío:

204 No Content

Comportamiento de upsert

Cada registro se identifica por el par (timestamp, name). Reenviar un registro con el mismo timestamp y nombre de serie sobrescribe el valor almacenado.

Los reintentos y los rellenos son idempotentes

Como la escritura es un upsert, puedes reintentar con seguridad una solicitud fallida o reenviar datos históricos: los duplicados se colapsan sobre la misma clave en lugar de acumularse.

Agrupación en lotes

Prefiere enviar muchos registros en una sola solicitud antes que un registro por solicitud. La agrupación en lotes reduce drásticamente los viajes de ida y vuelta y la sobrecarga. Al cargar el histórico, envíalo en bloques de muchos registros cada uno en lugar de transmitir un punto a la vez.

Errores

Los errores devuelven el estado HTTP más un cuerpo JSON con la forma { "detail": "..." }.

Estadomensaje detailSignificado
401Invalid access token. / Missing access token.La autenticación falló: revisa tu token.
404Data source not found.La fuente de datos indicada no existe. Créala primero en el panel.
409Data source is archived.La fuente de datos está archivada y no puede aceptar escrituras. Desarchívala o usa otra fuente.

Consulta Errores y códigos de estado para ver la referencia completa.
