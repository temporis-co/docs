Fuentes de datos

Una fuente de datos es donde viven tus observaciones en bruto. Esta página cubre el modelo de datos de Temporis y cómo enviar datos hacia él correctamente.

Creas una fuente de datos una sola vez en el panel y luego transmites observaciones hacia ella a través de la API a medida que ocurren.

El modelo de datos

Temporis almacena los datos como filas. Cada observación es un único triple: cuándo se midió algo, qué se midió y el número que mediste.

timestamp
Tiempo Unix en segundos (un entero). El momento en que se tomó la observación.
name
La serie a la que pertenece este valor; por ejemplo orders o revenue.
value
Un número. Los decimales están bien.

Una sola fuente de datos puede contener muchas series con nombre lado a lado. El campo name es lo que las mantiene separadas. Tus datos están aislados de los de cualquier otro cliente: una fuente de datos te pertenece solo a ti.

Crear una fuente de datos

Creas una fuente de datos en el panel, en Fuentes de datos. Dale un nombre y estará lista para recibir datos.

Los nombres siguen algunas reglas:

  • Empieza con una letra minúscula.
  • Usa solo letras minúsculas, dígitos y guiones bajos.
  • Hasta 48 caracteres.

Así, store_metrics y grid_load_eu son válidos; Store-Metrics y 2024data no lo son. Una fuente de datos recién creada está vacía hasta que le envíes observaciones.

Enviar observaciones

Agregas datos con el endpoint de ingesta. Cada llamada lleva el data_source de destino y una lista de records. Aquí enviamos algunas lecturas de orders a store_metrics.

curl -X POST https://api.temporis.co/v1/data_sources/ingest \
  -H "Authorization: Bearer ts_<token>" \
  -H "Content-Type: application/json" \
  -d '{
    "data_source": "store_metrics",
    "records": [
      { "timestamp": 1718870400, "name": "orders", "value": 42 },
      { "timestamp": 1718874000, "name": "orders", "value": 51 },
      { "timestamp": 1718877600, "name": "orders", "value": 47 }
    ]
  }'
import requests

requests.post(
    "https://api.temporis.co/v1/data_sources/ingest",
    headers={"Authorization": "Bearer ts_<token>"},
    json={
        "data_source": "store_metrics",
        "records": [
            {"timestamp": 1718870400, "name": "orders", "value": 42},
            {"timestamp": 1718874000, "name": "orders", "value": 51},
            {"timestamp": 1718877600, "name": "orders", "value": 47},
        ],
    },
)  # -> 204 No Content
await fetch("https://api.temporis.co/v1/data_sources/ingest", {
  method: "POST",
  headers: {
    "Authorization": "Bearer ts_<token>",
    "Content-Type": "application/json",
  },
  body: JSON.stringify({
    data_source: "store_metrics",
    records: [
      { timestamp: 1718870400, name: "orders", value: 42 },
      { timestamp: 1718874000, name: "orders", value: 51 },
      { timestamp: 1718877600, name: "orders", value: 47 },
    ],
  }),
}); // -> 204 No Content

Una ingesta exitosa devuelve HTTP 204 No Content con un cuerpo vacío: no hay nada que leer de vuelta, las filas simplemente se almacenan. Consulta la referencia de ingesta para conocer la solicitud completa y los detalles de errores.

Muchas series en una fuente

No estás limitado a una serie por fuente. Envía distintos valores de name al mismo data_source y coexistirán como series separadas. Una tienda podría seguir pedidos, ingresos y visitantes, todo en store_metrics.

{
  "data_source": "store_metrics",
  "records": [
    { "timestamp": 1718870400, "name": "orders",  "value": 42 },
    { "timestamp": 1718870400, "name": "revenue", "value": 1284.50 }
  ]
}

Mantener juntas las series relacionadas es conveniente y te permite pronosticarlas conjuntamente más adelante. Un perfil de datos es donde eliges cuáles de estos nombres leer realmente.

Actualizaciones y rellenos retroactivos

La clave de upsert es el par (timestamp, name). Enviar un registro cuyo timestamp y name ya existen sobrescribe el valor almacenado en lugar de añadir un duplicado.

Los rellenos retroactivos y los reintentos son seguros

Como las escrituras son idempotentes sobre (timestamp, name), puedes reproducir un lote tras un fallo de red, corregir una lectura equivocada o cargar meses de historia en cualquier orden: el resultado es siempre el mismo valor final por cubo.

Archivado

Cuando una fuente ya no recopila datos nuevos, puedes archivarla en el panel. El archivado detiene cualquier ingesta posterior: una solicitud dirigida a una fuente archivada devuelve HTTP 409 "Data source is archived."

El archivado no es destructivo. Los datos existentes permanecen en su lugar, y cualquier perfil de datos construido sobre la fuente sigue funcionando contra los datos ya almacenados.
