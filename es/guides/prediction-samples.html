Trabajar con muestras de predicción

/v1/predict devuelve numerosas trayectorias muestreadas. Esta guía las convierte en lo que realmente necesitas: un pronóstico puntual y una banda de confianza.

Por qué muchas trayectorias

Temporis genera pronósticos mediante muestreo: lee tu histórico y traza una continuación plausible, igual que un modelo de lenguaje continúa un texto. Cada extracción es un futuro posible, no el futuro. Pide count extracciones y obtienes count trayectorias independientes en predictions.

El sentido de tener muchas es la dispersión. Donde las trayectorias coinciden, el modelo está seguro; donde se abren en abanico, el futuro es genuinamente incierto. Esa dispersión es tu estimación de incertidumbre, y por eso una sola muestra rara vez es lo que quieres.

Ajusta count lo bastante alto como para resolver esa dispersión. Un puñado de extracciones da una imagen inestable y poco fiable; en algún punto del rango de 20 a 100 se resuelve bien la distribución para la mayoría de las series. Más es más suave, pero no gratis.

count cuesta tokens

La predicción se mide por token de modelo, y cada trayectoria genera un horizonte completo de tokens. Duplicar count duplica aproximadamente el coste de la llamada. Usa suficientes muestras para ver la dispersión con claridad, y luego detente.

Un pronóstico puntual

Para condensar las muestras en un solo número por marca de tiempo futura, agrega a través de las trayectorias en cada paso de tiempo. La mediana es un buen valor por defecto: es robusta frente a alguna trayectoria descontrolada ocasional; la media también funciona si la prefieres.

Las filas de data de cada predicción comparten las mismas marcas de tiempo en el mismo orden, así que puedes alinearlas por índice. Aquí pedimos 50 muestras del perfil hourly_orders y calculamos un pronóstico mediano de orders por marca de tiempo:

import os, requests
from statistics import median

token = os.environ["TEMPORIS_TOKEN"]

resp = requests.post(
    "https://api.temporis.co/v1/predict",
    headers={"Authorization": f"Bearer {token}"},
    json={"data_profile": "hourly_orders", "count": 50, "temperature": 1.0, "top_p": 0.9},
)
resp.raise_for_status()
body = resp.json()

# Each prediction's rows are in the same timestamp order, so transpose by index.
preds = body["predictions"]            # length == count (50)
rows = list(zip(*[p["data"] for p in preds]))  # group row i across all samples

point_forecast = []
for step in rows:
    ts = step[0]["timestamp"]          # shared timestamp for this step
    values = [row["orders"] for row in step]
    point_forecast.append({"timestamp": ts, "orders": median(values)})

for f in point_forecast[:3]:
    print(f["timestamp"], round(f["orders"], 1))

Bandas de confianza

La mediana te da una línea central; una banda te da el rango a su alrededor. Calcula cuantiles por marca de tiempo a través de las muestras: los percentiles 10, 50 y 90 son una elección habitual. El resultado son tres líneas: un borde inferior (p10), el centro (p50) y un borde superior (p90).

Lee la banda como "alrededor del 80 % de los futuros muestreados caen entre p10 y p90 en esta marca de tiempo". Una banda estrecha significa que el modelo está seguro; una banda que se ensancha a medida que miras más adelante es el modelo diciéndote honestamente que el futuro lejano es menos cierto que el cercano.

import os, requests
import numpy as np

token = os.environ["TEMPORIS_TOKEN"]

resp = requests.post(
    "https://api.temporis.co/v1/predict",
    headers={"Authorization": f"Bearer {token}"},
    json={"data_profile": "hourly_orders", "count": 50, "temperature": 1.0, "top_p": 0.9},
)
resp.raise_for_status()
body = resp.json()

preds = body["predictions"]
timestamps = [row["timestamp"] for row in preds[0]["data"]]

# Shape: (count, horizon) — one row per sample, one column per timestep.
matrix = np.array([[row["orders"] for row in p["data"]] for p in preds])

p10, p50, p90 = np.percentile(matrix, [10, 50, 90], axis=0)

band = [
    {"timestamp": ts, "p10": lo, "p50": mid, "p90": hi}
    for ts, lo, mid, hi in zip(timestamps, p10, p50, p90)
]
for b in band[:3]:
    print(b["timestamp"], round(b["p10"], 1), round(b["p50"], 1), round(b["p90"], 1))

Elegir count, temperature y top_p en la práctica

Tres perillas dan forma a las muestras, y hacen trabajos distintos:

  • count controla la resolución: con qué finura se muestrea la dispersión. No cambia la forma de la distribución, solo la claridad con que la ves.
  • temperature (mayor que 0) controla la dispersión: los valores más altos trazan futuros más diversos y de mayor amplitud; los más bajos agrupan las muestras cerca de la trayectoria más probable del modelo. Alrededor de 1.0 es neutral.
  • top_p (de 0 a 1) recorta las colas: bajarlo elimina los extremos improbables, así que obtienes menos saltos sorprendentes; 1.0 conserva la distribución completa.

Puntos de partida sensatos, según lo abrupta que sea tu serie:

Carácter de la seriecounttemperaturetop_p
Estable, suave (demanda constante)20–300.80.9
Típica501.00.9
Volátil, con picos (precios, tráfico a ráfagas)80–1001.10.95

Trátalos como un lugar por donde empezar. Dibuja una banda, observa si es demasiado estrecha como para ser honesta o tan ancha que resulta inútil, y ajusta primero temperature.

Ordenar por loss

A veces quieres una sola trayectoria representativa en lugar de una banda: para una demostración, una comprobación de sensatez o un rápido "qué aspecto tiene un futuro típico". Cada trayectoria lleva un loss, y un loss más bajo significa que el modelo consideró esa trayectoria más típica. Ordena por él y toma el principio de la lista:

# body is the parsed /v1/predict response
ranked = sorted(body["predictions"], key=lambda p: p["loss"])

most_typical = ranked[0]               # lowest loss = most representative path
for row in most_typical["data"][:3]:
    print(row["timestamp"], row["orders"])

# Or trim outliers: keep the most-typical 80% before computing a band.
keep = ranked[: int(len(ranked) * 0.8)]

Para un pronóstico real, prefiere la mediana y una banda antes que cualquier trayectoria individual: la trayectoria de menor loss es un futuro plausible, no un promedio de ellos. La ordenación es más útil para elegir un ejemplo limpio o filtrar valores atípicos.
