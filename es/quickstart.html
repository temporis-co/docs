Inicio rápido

Esta guía te lleva de una cuenta nueva a tu primer pronóstico. Crearás una clave de API, registrarás una fuente de datos, transmitirás algo de historial, construirás un perfil de datos y llamarás al endpoint de predicción. Calcula unos cinco minutos.

Dos superficies, una plataforma. Configurarás las cosas en el panel (claves, fuentes de datos, perfiles de datos, facturación) y realizarás el trabajo de datos y predicción a través de la API. Esta página recorre ambos.

  1. Crea un token de acceso

    En el panel, abre Tokens de acceso y crea uno. El token se muestra una sola vez: cópialo de inmediato. Tiene el aspecto ts_… y autentica cada solicitud a la API.

    Guárdalo como una variable de entorno para que los ejemplos siguientes funcionen tal cual:

    export TEMPORIS_TOKEN="ts_your_token_here"

  2. Añade un método de pago

    La predicción se cobra por token de uso, por lo que se requiere una tarjeta registrada antes de que /v1/predict se ejecute. Añade una en Facturación. Ingerir datos no requiere facturación: solo la predicción la requiere.

  3. Crea una fuente de datos

    En Fuentes de datos, crea una llamada store_metrics. Los nombres deben empezar con una letra minúscula y contener únicamente letras minúsculas, dígitos y guiones bajos. Una fuente de datos es un contenedor vacío hasta que le envías datos.

  4. Envía algo de historial

    Cada registro es una tripleta (timestamp, name, value). El timestamp es una hora Unix en segundos, name es la serie y value es el número. Aquí enviamos un puñado de lecturas horarias de orders; en la práctica, enviarías todo el historial real que tengas.

  5. Construye un perfil de datos

    De vuelta en el panel, abre Perfiles de datos y crea uno llamado hourly_orders. Apúntalo a la fuente de datos store_metrics, selecciona la serie orders y establece el intervalo en 3600 segundos (una hora). Los valores predeterminados para todo lo demás son un buen punto de partida.

    Un perfil necesita una base de historial antes de que Temporis pueda prepararlo para el servicio. Una vez que tiene suficientes datos, el panel lo marca como listo. Hasta entonces, las predicciones devuelven un error de "no listo": eso es lo esperado.

  6. Genera un pronóstico

    Ahora pídele al perfil lo que viene a continuación. count es cuántas trayectorias futuras independientes dibujar; temperature y top_p controlan qué tan arriesgado es el muestreo. 

    curl -X POST "https://api.temporis.co/v1/predict" \
  -H "Authorization: Bearer $TEMPORIS_TOKEN" \
  --json '{
    "data_profile": "hourly_orders",
    "count": 5,
    "temperature": 1.0,
    "top_p": 0.9
  }'

Qué acaba de pasar

Transmitiste observaciones a una fuente de datos, describiste cómo leerlas con un perfil de datos y dibujaste cinco futuros posibles desde /v1/predict. Cada predicción es una trayectoria completa; juntas forman una imagen probabilística de lo que viene a continuación. Cuanto menor sea el loss de una predicción, más típica consideró el modelo esa trayectoria.
