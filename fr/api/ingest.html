Ingérer des enregistrements

Envoyez des observations dans une source de données. La source de données doit déjà exister (créez-la dans le tableau de bord) et ne pas être archivée.

POST/v1/data_sources/ingest

Corps de la requête

Le corps nomme la source de données cible et transporte un lot d'enregistrements.

data_sourcestringRequis

Le nom d'une source de données existante et non archivée dans laquelle écrire.

recordsarrayRequis

Une liste d'enregistrements à stocker. Chaque enregistrement est une seule observation d'une série à un instant donné.

Champs d'un enregistrement

timestampintegerRequis

Temps Unix en secondes.

namestringRequis

Le nom de la série. Une source de données contient de nombreuses séries nommées.

valuenumberRequis

La valeur observée (un entier ou un nombre à virgule flottante).

Exemple de requête

Ceci écrit quatre observations d'orders dans la source de données store_metrics, en lisant le jeton depuis l'environnement :

curl https://api.temporis.co/v1/data_sources/ingest \
  -H "Authorization: Bearer $TEMPORIS_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "data_source": "store_metrics",
    "records": [
      { "timestamp": 1718841600, "name": "orders", "value": 42 },
      { "timestamp": 1718845200, "name": "orders", "value": 51 },
      { "timestamp": 1718848800, "name": "orders", "value": 47 },
      { "timestamp": 1718852400, "name": "orders", "value": 63 }
    ]
  }'

Réponse

En cas de succès, le point de terminaison renvoie 204 No Content avec un corps vide :

204 No Content

Comportement d'upsert

Chaque enregistrement est identifié par le couple (timestamp, name). Renvoyer un enregistrement avec le même timestamp et le même nom de série écrase la valeur stockée.

Les nouvelles tentatives et les remplissages rétroactifs sont idempotents

Parce que l'écriture est un upsert, vous pouvez relancer en toute sécurité une requête échouée ou renvoyer des données historiques — les doublons se rabattent sur la même clé au lieu de s'accumuler.

Regroupement par lots

Préférez envoyer de nombreux enregistrements en une seule requête plutôt qu'un enregistrement par requête. Le regroupement par lots réduit considérablement les allers-retours et la surcharge. Lors du chargement de l'historique, envoyez-le par fragments de nombreux enregistrements chacun plutôt que de diffuser un point à la fois.

Erreurs

Les erreurs renvoient le statut HTTP plus un corps JSON de la forme { "detail": "..." }.

Statutmessage detailSignification
401Invalid access token. / Missing access token.L'authentification a échoué — vérifiez votre jeton.
404Data source not found.La source de données nommée n'existe pas. Créez-la d'abord dans le tableau de bord.
409Data source is archived.La source de données est archivée et ne peut pas accepter d'écritures. Désarchivez-la ou utilisez une autre source.

Consultez Erreurs & codes de statut pour la référence complète.
