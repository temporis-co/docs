Travailler avec les échantillons de prédiction

/v1/predict renvoie de nombreuses trajectoires échantillonnées. Ce guide les transforme en ce dont vous avez réellement besoin : une prévision ponctuelle et une bande de confiance.

Pourquoi de nombreuses trajectoires

Temporis génère des prévisions par échantillonnage — il lit votre historique et tire une continuation plausible, à la manière dont un modèle de langage prolonge un texte. Chaque tirage est un futur possible, pas le futur. Demandez count tirages et vous obtenez count trajectoires indépendantes dans predictions.

L'intérêt d'en avoir plusieurs est la dispersion. Là où les trajectoires s'accordent, le modèle est confiant ; là où elles s'évasent, le futur est véritablement incertain. Cette dispersion est votre estimation d'incertitude — c'est pourquoi un seul échantillon est rarement ce que vous voulez.

Réglez count assez haut pour résoudre cette dispersion. Une poignée de tirages donne une image instable et peu fiable ; quelque part dans la plage de 20 à 100 résout bien la distribution pour la plupart des séries. Plus est plus lisse mais pas gratuit.

Count coûte des jetons

La prédiction est facturée par jeton du modèle, et chaque trajectoire génère un horizon complet de jetons. Doubler count double à peu près le coût de l'appel. Utilisez assez d'échantillons pour voir clairement la dispersion, puis arrêtez.

Une prévision ponctuelle

Pour réduire les échantillons à un seul nombre par timestamp futur, agrégez à travers les trajectoires à chaque pas de temps. La médiane est une bonne valeur par défaut — elle est robuste face à la trajectoire occasionnellement aberrante ; la moyenne fonctionne aussi si vous la préférez.

Les lignes data de chaque prédiction partagent les mêmes timestamps dans le même ordre, donc vous pouvez les aligner par index. Ici, nous demandons 50 échantillons du profil hourly_orders et calculons une prévision médiane d'orders par timestamp :

import os, requests
from statistics import median

token = os.environ["TEMPORIS_TOKEN"]

resp = requests.post(
    "https://api.temporis.co/v1/predict",
    headers={"Authorization": f"Bearer {token}"},
    json={"data_profile": "hourly_orders", "count": 50, "temperature": 1.0, "top_p": 0.9},
)
resp.raise_for_status()
body = resp.json()

# Each prediction's rows are in the same timestamp order, so transpose by index.
preds = body["predictions"]            # length == count (50)
rows = list(zip(*[p["data"] for p in preds]))  # group row i across all samples

point_forecast = []
for step in rows:
    ts = step[0]["timestamp"]          # shared timestamp for this step
    values = [row["orders"] for row in step]
    point_forecast.append({"timestamp": ts, "orders": median(values)})

for f in point_forecast[:3]:
    print(f["timestamp"], round(f["orders"], 1))

Bandes de confiance

La médiane vous donne une ligne centrale ; une bande vous donne la plage autour d'elle. Calculez des quantiles par timestamp à travers les échantillons — les 10e, 50e et 90e centiles sont un choix courant. Le résultat est trois lignes : un bord bas (p10), le centre (p50) et un bord haut (p90).

Lisez la bande comme « environ 80 % des futurs échantillonnés se situent entre p10 et p90 à ce timestamp ». Une bande étroite signifie que le modèle est confiant ; une bande qui s'élargit à mesure que vous regardez plus loin est le modèle vous disant honnêtement que le futur lointain est moins certain que le futur proche.

import os, requests
import numpy as np

token = os.environ["TEMPORIS_TOKEN"]

resp = requests.post(
    "https://api.temporis.co/v1/predict",
    headers={"Authorization": f"Bearer {token}"},
    json={"data_profile": "hourly_orders", "count": 50, "temperature": 1.0, "top_p": 0.9},
)
resp.raise_for_status()
body = resp.json()

preds = body["predictions"]
timestamps = [row["timestamp"] for row in preds[0]["data"]]

# Shape: (count, horizon) — one row per sample, one column per timestep.
matrix = np.array([[row["orders"] for row in p["data"]] for p in preds])

p10, p50, p90 = np.percentile(matrix, [10, 50, 90], axis=0)

band = [
    {"timestamp": ts, "p10": lo, "p50": mid, "p90": hi}
    for ts, lo, mid, hi in zip(timestamps, p10, p50, p90)
]
for b in band[:3]:
    print(b["timestamp"], round(b["p10"], 1), round(b["p50"], 1), round(b["p90"], 1))

Choisir count, temperature, top_p en pratique

Trois boutons façonnent les échantillons, et ils font des travaux différents :

  • count contrôle la résolution — la finesse avec laquelle la dispersion est échantillonnée. Il ne change pas la forme de la distribution, seulement la clarté avec laquelle vous la voyez.
  • temperature (supérieure à 0) contrôle la dispersion — des valeurs plus élevées tirent des futurs plus divers et de plus large amplitude ; des valeurs plus faibles regroupent les échantillons près du chemin le plus probable du modèle. Environ 1.0 est neutre.
  • top_p (0 à 1) rogne les queues — le baisser coupe les extrêmes improbables, donc vous obtenez moins de sauts surprenants ; 1.0 conserve toute la distribution.

Points de départ raisonnables, selon le degré de nervosité de votre série :

Caractère de la sériecounttemperaturetop_p
Stable, lisse (demande régulière)20–300.80.9
Typique501.00.9
Volatile, en pics (prix, trafic en rafales)80–1001.10.95

Considérez-les comme un point de départ. Tracez une bande, regardez si elle est trop serrée pour être honnête ou si large qu'elle est inutile, et ajustez d'abord temperature.

Classer par loss

Parfois, vous voulez un seul chemin représentatif plutôt qu'une bande — pour une démo, une vérification de cohérence ou un rapide « à quoi ressemble un futur typique ». Chaque trajectoire porte une loss, et une loss plus faible signifie que le modèle a jugé ce chemin plus typique. Triez par elle et prenez le début de la liste :

# body is the parsed /v1/predict response
ranked = sorted(body["predictions"], key=lambda p: p["loss"])

most_typical = ranked[0]               # lowest loss = most representative path
for row in most_typical["data"][:3]:
    print(row["timestamp"], row["orders"])

# Or trim outliers: keep the most-typical 80% before computing a band.
keep = ranked[: int(len(ranked) * 0.8)]

Pour une prévision réelle, préférez la médiane et une bande à toute trajectoire unique — le chemin à plus faible loss est un futur plausible, pas une moyenne de ceux-ci. Le classement est surtout utile pour choisir un exemple propre ou filtrer les valeurs aberrantes.
