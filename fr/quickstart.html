Démarrage rapide

Ce guide vous fait passer d'un compte tout neuf à votre première prévision. Vous allez créer une clé d'API, enregistrer une source de données, y diffuser un peu d'historique, construire un profil de données et appeler le point de terminaison de prédiction. Comptez environ cinq minutes.

Deux surfaces, une seule plateforme. Vous configurerez les choses dans le tableau de bord (clés, sources de données, profils de données, facturation) et vous effectuerez le travail de données et de prévision via l'API. Cette page parcourt les deux.

  1. Créez un jeton d'accès

    Dans le tableau de bord, ouvrez Jetons d'accès et créez-en un. Le jeton est affiché une seule fois — copiez-le immédiatement. Il ressemble à ts_… et authentifie chaque requête de l'API.

    Stockez-le dans une variable d'environnement pour que les exemples ci-dessous fonctionnent tels quels :

    export TEMPORIS_TOKEN="ts_your_token_here"

  2. Ajoutez un moyen de paiement

    La prédiction est facturée par jeton d'utilisation, une carte enregistrée est donc requise avant que /v1/predict ne s'exécute. Ajoutez-en une sous Facturation. L'ingestion de données ne nécessite pas de facturation — seule la prévision en a besoin.

  3. Créez une source de données

    Sous Sources de données, créez-en une nommée store_metrics. Les noms doivent commencer par une lettre minuscule et ne contenir que des lettres minuscules, des chiffres et des traits de soulignement. Une source de données est un conteneur vide tant que vous ne lui envoyez pas de données.

  4. Envoyez un peu d'historique

    Chaque enregistrement est un triplet (timestamp, name, value). Le timestamp est un temps Unix en secondes, name est la série et value est le nombre. Ici, nous envoyons quelques relevés horaires d'orders ; en pratique, vous enverriez autant d'historique réel que vous en avez.

  5. Construisez un profil de données

    De retour dans le tableau de bord, ouvrez Profils de données et créez-en un nommé hourly_orders. Pointez-le vers la source de données store_metrics, sélectionnez la série orders et réglez l'intervalle sur 3600 secondes (une heure). Les valeurs par défaut pour tout le reste constituent un bon point de départ.

    Un profil a besoin d'une base d'historique avant que Temporis puisse le préparer au service. Une fois qu'il dispose de suffisamment de données, le tableau de bord le marque comme prêt. Jusque-là, les prédictions renvoient une erreur « not ready » — c'est attendu.

  6. Générez une prévision

    Demandez maintenant au profil ce qui vient ensuite. count est le nombre de trajectoires futures indépendantes à tirer ; temperature et top_p contrôlent l'audace de l'échantillonnage.

    "temperature": 1.0,

Ce qui vient de se passer

Vous avez diffusé des observations dans une source de données, décrit comment les lire avec un profil de données et tiré cinq futurs possibles de /v1/predict. Chaque prédiction est une trajectoire complète ; ensemble, elles forment une image probabiliste de ce qui vient ensuite. Plus la loss d'une prédiction est faible, plus le modèle a jugé cette trajectoire typique.
