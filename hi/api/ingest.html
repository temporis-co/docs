रिकॉर्ड इन्जेस्ट करें

किसी data source में observations भेजें। data source पहले से मौजूद होना चाहिए (इसे dashboard में बनाएं) और archived नहीं होना चाहिए।

POST/v1/data_sources/ingest

Request body

Body लक्षित data source का नाम देती है और records का एक batch ले जाती है।

data_sourcestringआवश्यक

किसी मौजूदा, गैर-archived data source का नाम जिसमें लिखना है।

recordsarrayआवश्यक

store करने के लिए records की सूची। प्रत्येक record एक समय पर एक series का एकल observation है।

Record fields

timestampintegerआवश्यक

Unix समय, सेकंड में।

namestringआवश्यक

series का नाम। एक data source कई named series रखता है।

valuenumberआवश्यक

observed मान (integer या float)।

Response

सफलता पर endpoint खाली body के साथ 204 No Content लौटाता है:

204 No Content

Upsert व्यवहार

प्रत्येक record की key जोड़ी (timestamp, name) होती है। उसी timestamp और series नाम के साथ record दोबारा भेजने पर stored मान overwrite हो जाता है।

Retries और backfills idempotent हैं

चूंकि write एक upsert है, आप किसी विफल request को सुरक्षित रूप से दोबारा भेज सकते हैं या ऐतिहासिक data फिर से भेज सकते हैं — duplicates जमा होने के बजाय उसी key पर समाहित हो जाते हैं।

Batching

प्रति request एक record भेजने के बजाय एक ही request में कई records भेजना बेहतर है। Batching round trips और overhead को काफी कम कर देता है। history लोड करते समय, इसे एक बार में एक point स्ट्रीम करने के बजाय प्रत्येक में कई records के chunks में भेजें।

Errors

Errors HTTP status के साथ { "detail": "..." } रूप की एक JSON body लौटाते हैं।

Statusdetail संदेशअर्थ
401Invalid access token. / Missing access token.Authentication विफल — अपना token जांचें।
404Data source not found.नामित data source मौजूद नहीं है। पहले इसे dashboard में बनाएं।
409Data source is archived.data source archived है और writes स्वीकार नहीं कर सकता। इसे unarchive करें या दूसरा source उपयोग करें।

पूरे रेफरेंस के लिए Errors & status codes देखें।
