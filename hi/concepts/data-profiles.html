Data profiles

data profile वह नुस्खा है जो Temporis को बताता है कि किसी data source को कैसे पढ़ना है: कौन-सी series, किस cadence पर, कितना इतिहास, और इसे कैसे साफ़ करना है। यही वह चीज़ है जिसके विरुद्ध आप forecast करते हैं।

एक profile क्या परिभाषित करता है

एक data source raw rows रखता है। एक profile उन rows को एक साफ़, नियमित रूप से दूरी वाले signal में बदलता है जिसे model पढ़ सके — series चुनना, observations को एक time grid पर snap करना, यह तय करना कि कितना इतिहास उपयोग करना है, और gaps को व्यवस्थित करना। जब आप predict call करते हैं, तो आप किसी source को नहीं, बल्कि एक profile को नाम देते हैं।

आप profiles को dashboard में बनाते और संपादित करते हैं। यहाँ का उदाहरण hourly_orders है: यह store_metrics से orders series को एक-घंटे की grid पर पढ़ता है (interval 3600)।

डेटा का चयन

Source और series

दो settings चुनती हैं कि profile क्या पढ़ता है:

  • data_source — किस data source से पढ़ना है।
  • names — उस source के भीतर एक या अधिक series names।

एक अकेला name listing करने पर एक univariate forecast मिलता है। कई listing करने पर — मान लें orders और visitors — profile multivariate बन जाता है: series को संयुक्त रूप से forecast किया जाता है, ताकि model उनके बीच के संबंध का उपयोग कर सके।

time grid

असली observations शायद ही कभी एकदम समान cadence पर आते हैं, इसलिए एक profile उन्हें एक नियमित grid पर resample करता है।

interval सेकंड्स में प्रत्येक grid bucket का आकार है। interval 3600 के साथ, observations को घंटेवार buckets में snap किया जाता है। grid आपके सबसे हाल के observation पर anchored होती है और वहाँ से पीछे की ओर बढ़ती है, ताकि नवीनतम डेटा हमेशा grid पर साफ़-साफ़ बैठे।

model हर बिंदु को एक-एक करके नहीं पढ़ता। यह patches में काम करता है — series के निश्चित-आकार के टुकड़े:

  • patch_len — कितने बिंदु एक patch बनाते हैं।
  • patch_spacing — जिन बिंदुओं का उपयोग होता है उनके बीच का stride, एक प्रकार का downsampling।
  • num_patches — context के रूप में यह अधिकतम कितने patches के इतिहास का उपयोग करेगा।
  • min_patches — एक prediction देने के लिए कम से कम कितने patches का इतिहास आवश्यक है।

सहज रूप से: model जो context देखता है वह मोटे तौर पर हाल के इतिहास के num_patches patches होता है। यदि किसी profile के पास min_patches से कम patches उपलब्ध हों, तो वह serve नहीं कर सकता और आपको HTTP 409 "Not enough data for this data profile." मिलता है।

Spacing आपके forecast का resolution भी तय करता है। forecast बिंदुओं के बीच प्रभावी step interval × patch_spacing सेकंड्स होता है। interval 3600 और patch_spacing 1 के साथ, predictions एक घंटे की दूरी पर बैठती हैं।

series को साफ़ करना

resampling के बाद, कुछ buckets में कोई observation नहीं हो सकता। दो settings तय करती हैं कि इसे कैसे संभाला जाए:

  • fill_limit — लगातार खाली buckets का सबसे लंबा क्रम जिसे forward-fill किया जाएगा (अंतिम ज्ञात value को आगे ले जाते हुए)। इससे लंबा gap वास्तव में missing माना जाता है और उन rows को इसके बजाय हटा दिया जाता है।
  • exclude_window — मास्क करने के लिए दिन के समय की एक आवर्ती window, जो एक timezone के साथ-साथ एक start और stop के रूप में दी जाती है। इसे उस अपेक्षित dead time के लिए उपयोग करें, जैसे रात के घंटे जब कोई market बंद रहता है, ताकि वह शांत अवधि series को विकृत न करे।

Levels बनाम growth

कुछ series को उनके level से सबसे अच्छी तरह model किया जाता है; अन्य को उनके सापेक्ष परिवर्तन से। use_diffs setting (log-diff transform) model को raw value के बजाय log growth — एक step से अगले तक का आनुपातिक परिवर्तन — पर काम करने के लिए switch कर देती है।

इसे उन series के लिए चालू करें जो गुणात्मक रूप से चलती हैं या एक विस्तृत range में मज़बूती से trend करती हैं: revenue, prices, कुछ भी जहाँ 10% का बदलाव एक absolute बदलाव से अधिक मायने रखता है। इसे उन series के लिए बंद रखें जो एक bounded, मोटे तौर पर stationary range में रहती हैं, जहाँ raw level पहले से ही predict करने के लिए स्वाभाविक चीज़ है।

Readiness

profile बनाते ही उपयोग करने योग्य नहीं होता। Temporis पहले normalization statistics की गणना करता है — आपके डेटा के सामान्य फैलाव को सीखते हुए ताकि वह values को उस range में scale कर सके जिसकी model अपेक्षा करता है। जब तक उसके पास ऐसा करने के लिए पर्याप्त डेटा न हो, profile तैयार नहीं होता, और dashboard उसकी readiness state दिखाता है।

ऐसे profile के विरुद्ध predict करना जो तैयार नहीं है, HTTP 404 "Data profile is not ready to serve." लौटाता है। तब तक प्रतीक्षा करें जब तक dashboard profile को तैयार न बताए, फिर predict call करें।
