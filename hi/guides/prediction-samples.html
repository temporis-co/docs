prediction samples के साथ काम करना

/v1/predict अनेक sampled प्रक्षेपपथ लौटाता है। यह गाइड उन्हें उन चीज़ों में बदलती है जिनकी आपको वास्तव में ज़रूरत है: एक point forecast और एक confidence band।

अनेक प्रक्षेपपथ क्यों

Temporis sampling द्वारा पूर्वानुमान उत्पन्न करता है — यह आपका इतिहास पढ़ता है और एक प्रशंसनीय निरंतरता खींचता है, ठीक वैसे ही जैसे एक भाषा मॉडल पाठ को आगे बढ़ाता है। प्रत्येक draw एक संभावित भविष्य है, न कि वह भविष्य। count draws माँगें और आपको predictions में count स्वतंत्र प्रक्षेपपथ मिलते हैं।

अनेक होने का मतलब है फैलाव। जहाँ प्रक्षेपपथ सहमत होते हैं, वहाँ मॉडल आश्वस्त है; जहाँ वे पंखे की तरह फैलते हैं, वहाँ भविष्य वास्तव में अनिश्चित है। वह फैलाव आपका अनिश्चितता आकलन है — इसीलिए एक अकेला sample शायद ही कभी वह होता है जो आप चाहते हैं।

count को उस फैलाव को सुलझाने के लिए पर्याप्त ऊँचा सेट करें। मुट्ठी भर draws एक झटकेदार, अविश्वसनीय तस्वीर देते हैं; 20 से 100 की सीमा में कहीं अधिकांश series के लिए वितरण को अच्छी तरह सुलझा देता है। अधिक अधिक चिकना होता है पर मुफ़्त नहीं।

Count tokens में खर्च होता है

पूर्वानुमान प्रति मॉडल token मीटर किया जाता है, और प्रत्येक प्रक्षेपपथ tokens का एक पूरा क्षितिज उत्पन्न करता है। count को दोगुना करने से कॉल की लागत लगभग दोगुनी हो जाती है। फैलाव को साफ़ देखने के लिए पर्याप्त samples का उपयोग करें, फिर रुक जाएँ।

एक point forecast

samples को प्रति भविष्य timestamp एक संख्या में समेटने के लिए, प्रत्येक timestep पर प्रक्षेपपथों के आर-पार समुच्चय करें। median एक अच्छा डिफ़ॉल्ट है — यह कभी-कभार के जंगली प्रक्षेपपथ के प्रति मज़बूत है; अगर आप पसंद करें तो mean भी काम करता है।

प्रत्येक prediction की data पंक्तियाँ समान timestamps को समान क्रम में साझा करती हैं, इसलिए आप उन्हें index से पंक्तिबद्ध कर सकते हैं। यहाँ हम hourly_orders profile से 50 samples माँगते हैं और प्रति timestamp एक median orders forecast गणना करते हैं:

import os, requests
from statistics import median

token = os.environ["TEMPORIS_TOKEN"]

resp = requests.post(
    "https://api.temporis.co/v1/predict",
    headers={"Authorization": f"Bearer {token}"},
    json={"data_profile": "hourly_orders", "count": 50, "temperature": 1.0, "top_p": 0.9},
)
resp.raise_for_status()
body = resp.json()

# Each prediction's rows are in the same timestamp order, so transpose by index.
preds = body["predictions"]            # length == count (50)
rows = list(zip(*[p["data"] for p in preds]))  # group row i across all samples

point_forecast = []
for step in rows:
    ts = step[0]["timestamp"]          # shared timestamp for this step
    values = [row["orders"] for row in step]
    point_forecast.append({"timestamp": ts, "orders": median(values)})

for f in point_forecast[:3]:
    print(f["timestamp"], round(f["orders"], 1))

Confidence bands

median आपको एक केंद्र रेखा देता है; एक band आपको उसके चारों ओर की सीमा देता है। samples के आर-पार प्रति-timestamp quantiles गणना करें — 10वाँ, 50वाँ, और 90वाँ पर्सेंटाइल एक आम विकल्प हैं। परिणाम तीन रेखाएँ हैं: एक निचला किनारा (p10), केंद्र (p50), और एक ऊपरी किनारा (p90)।

band को इस तरह पढ़ें "इस timestamp पर लगभग 80% sampled भविष्य p10 और p90 के बीच आते हैं।" एक संकरा band मतलब मॉडल आश्वस्त है; एक band जो आगे देखने पर चौड़ा होता जाता है, वह मॉडल का ईमानदारी से आपको यह बताना है कि दूर का भविष्य निकट के भविष्य से कम निश्चित है।

import os, requests
import numpy as np

token = os.environ["TEMPORIS_TOKEN"]

resp = requests.post(
    "https://api.temporis.co/v1/predict",
    headers={"Authorization": f"Bearer {token}"},
    json={"data_profile": "hourly_orders", "count": 50, "temperature": 1.0, "top_p": 0.9},
)
resp.raise_for_status()
body = resp.json()

preds = body["predictions"]
timestamps = [row["timestamp"] for row in preds[0]["data"]]

# Shape: (count, horizon) — one row per sample, one column per timestep.
matrix = np.array([[row["orders"] for row in p["data"]] for p in preds])

p10, p50, p90 = np.percentile(matrix, [10, 50, 90], axis=0)

band = [
    {"timestamp": ts, "p10": lo, "p50": mid, "p90": hi}
    for ts, lo, mid, hi in zip(timestamps, p10, p50, p90)
]
for b in band[:3]:
    print(b["timestamp"], round(b["p10"], 1), round(b["p50"], 1), round(b["p90"], 1))

व्यवहार में count, temperature, top_p चुनना

तीन घुंडियाँ samples को आकार देती हैं, और वे अलग-अलग काम करती हैं:

  • count विभेदन को नियंत्रित करता है — फैलाव को कितनी बारीकी से sample किया जाता है। यह वितरण के आकार को नहीं बदलता, केवल इतना कि आप उसे कितना साफ़ देखते हैं।
  • temperature (0 से अधिक) फैलाव को नियंत्रित करता है — ऊँचे मान अधिक विविध, अधिक विस्तृत-सीमा वाले भविष्य खींचते हैं; निचले मान samples को मॉडल के सबसे-संभावित पथ के पास संकेंद्रित करते हैं। लगभग 1.0 तटस्थ है।
  • top_p (0 से 1) पूँछों को काटता है — इसे घटाने से असंभाव्य चरम कट जाते हैं, इसलिए आपको कम चौंकाने वाली छलांगें मिलती हैं; 1.0 पूरे वितरण को रखता है।

समझदार शुरुआती बिंदु, इस पर निर्भर कि आपकी series कितनी झटकेदार है:

series का स्वभावcounttemperaturetop_p
स्थिर, चिकनी (एकसमान माँग)20–300.80.9
सामान्य501.00.9
अस्थिर, नुकीली (कीमतें, झटकेदार ट्रैफ़िक)80–1001.10.95

इन्हें शुरुआत की जगह मानें। एक band खींचें, देखें कि क्या यह ईमानदार होने के लिए बहुत कसा हुआ है या इतना चौड़ा है कि बेकार है, और सबसे पहले temperature को समायोजित करें।

loss के अनुसार रैंकिंग

कभी-कभी आप एक band के बजाय एक अकेला प्रतिनिधि पथ चाहते हैं — किसी डेमो, किसी समझदारी जाँच, या एक त्वरित "एक सामान्य भविष्य कैसा दिखता है" के लिए। प्रत्येक प्रक्षेपपथ एक loss रखता है, और कम loss का मतलब है कि मॉडल ने उस पथ को अधिक सामान्य माना। उसके अनुसार छाँटें और सूची के सामने वाले हिस्से को लें:

# body is the parsed /v1/predict response
ranked = sorted(body["predictions"], key=lambda p: p["loss"])

most_typical = ranked[0]               # lowest loss = most representative path
for row in most_typical["data"][:3]:
    print(row["timestamp"], row["orders"])

# Or trim outliers: keep the most-typical 80% before computing a band.
keep = ranked[: int(len(ranked) * 0.8)]

एक वास्तविक पूर्वानुमान के लिए, किसी एक अकेले प्रक्षेपपथ के बजाय median और एक band को प्राथमिकता दें — सबसे-कम-loss वाला पथ एक संभावित भविष्य है, उनका औसत नहीं। रैंकिंग एक साफ़ उदाहरण चुनने या outliers को छानने के लिए सबसे उपयोगी है।
