認証

すべての Temporis API リクエストは、ベアラートークンで認証します。ここでは、トークンの作成方法、送信方法、安全な管理方法を説明します。

アクセストークン

アクセストークンは、ダッシュボードの Access Tokens から作成します。トークンは ts_ を接頭辞とする長いランダムな文字列です。

完全なトークンは、作成した瞬間に一度だけ表示されます。Temporis はサーバー側でそのハッシュのみを保存し、トークン自体は決して保存しません。つまり、紛失したトークンは復元できません。紛失した場合は、そのトークンを削除して新しいものを作成してください。

トークンはパスワードと同様に扱う

あなたのトークンを持つ者は誰でもデータを取り込み、予測を実行できます。そして予測はあなたのアカウントに課金されます。それに応じて厳重に保護してください。

リクエストの認証

すべてのリクエストで、Bearer スキームを使い、Authorization ヘッダーにトークンを送信します:

Authorization: Bearer ts_your_token_here

本ドキュメント全体の例では、トークンを TEMPORIS_TOKEN 環境変数から読み取るため、シークレットがソースコードに現れることはありません。シェルで一度エクスポートしてください:

export TEMPORIS_TOKEN="ts_your_token_here"

そして、ヘッダーとして付与します:

curl https://api.temporis.co/v1/predict \
  -H "Authorization: Bearer $TEMPORIS_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{ "data_profile": "hourly_orders", "count": 5, "temperature": 1.0, "top_p": 0.9 }'
import os, requests

token = os.environ["TEMPORIS_TOKEN"]

resp = requests.post(
    "https://api.temporis.co/v1/predict",
    headers={"Authorization": f"Bearer {token}"},
    json={"data_profile": "hourly_orders", "count": 5, "temperature": 1.0, "top_p": 0.9},
)
resp.raise_for_status()
const token = process.env.TEMPORIS_TOKEN;

const resp = await fetch("https://api.temporis.co/v1/predict", {
  method: "POST",
  headers: {
    "Authorization": `Bearer ${token}`,
    "Content-Type": "application/json",
  },
  body: JSON.stringify({ data_profile: "hourly_orders", count: 5, temperature: 1.0, top_p: 0.9 }),
});

認証に失敗した場合

Authorization ヘッダーが欠落または不正である場合、もしくはトークンが不明な場合、API は HTTP 401 と、何が問題だったかを示す JSON ボディを返します:

{ "detail": "Invalid access token." }

ヘッダーが欠落または不正な場合は、代わりに { "detail": "Missing access token." } を返します。いずれも 401 です。レスポンスの完全な一覧については、エラーとステータスコードを参照してください。

トークンを安全に保つ

いくつかの習慣により、トークンが不正な手に渡るのを防げます:

  • トークンを決してソース管理にコミットしないこと。また、クライアントサイドのコードに含めないこと。ブラウザやモバイルアプリは、そのソースをユーザーに配布します。
  • トークンは環境変数やシークレットマネージャーから読み取る。 ログ、スクリーンショット、チケットには含めないでください。
  • サービスごとに別々のトークンを作成することで、他に影響を与えずに 1 つだけを失効できます。
  • ダッシュボードでトークンを削除すれば即座に失効します。 削除は即時に有効になり、そのトークンを使う次のリクエストは 401 を受け取ります。
