エラーとステータスコード

Temporis が成功と失敗をどのように報告するか、各ステータスの意味、そしてその扱い方を解説します。

レスポンスの形式

レスポンスの形式は、何が起こったかによって異なります:

  • 取り込みが成功すると、ボディなしの 204 No Content を返します。
  • 予測が成功すると、JSON ボディとともに 200 を返します。
  • あらゆるエラーは、該当する HTTP ステータスと、{ "detail": "..." } という形式の JSON ボディを返します。
{ "detail": "Data profile not found." }

ステータスコード

ステータス意味対象
200 OK予測が成功しました。ボディにサンプルが含まれます。predict
204 No Content取り込みが成功しました。ボディはありません。ingest
400 Bad Request予測パラメータが不正です。predict
401 Unauthorizedアクセストークンが欠落または無効です。すべてのリクエスト
402 Payment Required登録済みのカードがない、または支払いに問題があります。predict
404 Not Foundデータソースが見つからない(ingest)、データプロファイルが見つからない、または提供準備ができていない(predict)。ingest, predict
409 Conflictデータソースがアーカイブ済み(ingest)、データが不足(predict)。ingest, predict
502 Bad Gateway予測が失敗しました。一時的なサーバー側のエラーです。predict

エラーの処理

ほとんどの失敗は、次に取るべき明確なアクションに対応します:

  • 401 — Authorization ヘッダーが存在し、トークンが有効であることを確認してください。認証を参照。
  • 402 — ダッシュボードの Billing でクレジットカードを追加または修正してから、再試行してください。
  • 404 — データソースまたはデータプロファイルが存在し、プロファイルが提供準備できていることを確認してください。
  • predict での 409 — プロファイルが最小要件を満たすまで履歴をさらに取り込んでから、再試行してください。
  • ingest での 409 — データソースがアーカイブされています。アーカイブを解除するか、別のソースに書き込んでください。
  • 502 — 一時的なものです。指数バックオフで再試行してください。

ステータスコードによる単純な分岐で、これらすべてに対応できます:

import time, requests

def predict(payload, token, retries=3):
    for attempt in range(retries):
        resp = requests.post(
            "https://api.temporis.co/v1/predict",
            headers={"Authorization": f"Bearer {token}"},
            json=payload,
        )
        if resp.status_code == 200:
            return resp.json()
        if resp.status_code == 502:
            time.sleep(2 ** attempt)   # transient: back off and retry
            continue
        detail = resp.json().get("detail", "")
        if resp.status_code == 402:
            raise RuntimeError(f"Billing: {detail} — add a card in the dashboard.")
        if resp.status_code == 409:
            raise RuntimeError(f"Not ready: {detail} — ingest more history.")
        resp.raise_for_status()        # 400 / 401 / 404: fix the request
    raise RuntimeError("Prediction failed after retries.")

リトライ

適切なものを再試行する

502 は一時的なものなので、再試行しても安全です。取り込みは冪等です(書き込みは (timestamp, name) キーでアップサートされます)。そのため、失敗した取り込みの再試行は常に安全です。4xx レスポンスを闇雲に再試行するのは避け、まずリクエストを修正してください。
