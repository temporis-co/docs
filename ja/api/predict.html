予測の作成

データプロファイルから 1 つ以上のサンプリングされた未来を引き出します。登録済みのクレジットカードが必要です。

POST/v1/predict

リクエストボディ

データプロファイル名と、取得したいサンプルの形状を指定します。

data_profilestring必須

あなたのプロファイルのいずれかの名前、または temporis/polymarket のような公開プロファイル。

countinteger必須

引き出す独立した未来の軌跡の数。1 以上である必要があります。

temperaturenumber必須

サンプリング温度。0 より大きい値です。高くするとサンプルが広く分散し、低くすると最も可能性の高い経路の近くに保たれます。

top_pnumber必須

核サンプリング(nucleus sampling)のカットオフ。0 から 1 の範囲です。低くすると可能性の低い裾野が切り捨てられ、1.0 では分布全体が保持されます。

count、temperature、top_p については、予測で詳しく説明しています。

リクエスト例

以下は、中立的な温度で hourly_orders プロファイルから 5 つの軌跡を引き出す例です:

curl https://api.temporis.co/v1/predict \
  -H "Authorization: Bearer $TEMPORIS_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "data_profile": "hourly_orders",
    "count": 5,
    "temperature": 1.0,
    "top_p": 0.9
  }'

レスポンスフィールド

リクエストが成功すると、このボディとともに HTTP 200 が返されます。

request_idstring

この予測リクエストの一意の識別子。

data_profilestring

リクエストしたプロファイル名をそのまま返します。

predict_frominteger

Unix 秒。予測が続きを開始する、最後に観測されたタイムスタンプです。すべての軌跡はここから前へ進みます。

predictionsarray

サンプリングされた軌跡。長さはリクエストした count と等しくなります。

Prediction オブジェクト

lossnumber

値が低いほど、モデルがこの軌跡をより典型的だと判断したことを意味します。代表的なサンプルをランク付けしたり選択したりするのに使えます。

dataarray

予測の各行。各行には timestamp と、プロファイル内の系列名ごとに 1 つのフィールドが含まれます。

公開データプロファイル

公開プロファイル名を data_profile として渡すことで、自分で何の準備もせずに共有データセットを予測できます。これらは Temporis が管理しており、すぐに利用できます:

{ "data_profile": "temporis/polymarket", "count": 3, "temperature": 1.0, "top_p": 0.9 }

エラー

エラーは、HTTP ステータスと、{ "detail": "..." } という形式の JSON ボディを返します。

ステータスdetail メッセージ意味
400Invalid prediction parameters.count が 1 未満、temperature が 0 以下など、パラメータが範囲外です。
402Add a credit card before using the API. / There is a payment issue. Add a new credit card.有効な課金設定がありません。ダッシュボードでカードを追加または修正してください。
404Data profile not found.その名前のプロファイルは存在しません。
404Data profile is not ready to serve.プロファイルは存在しますが、準備が完了していません。
409Not enough data for this data profile.プロファイルの履歴が、必要な最小量に達していません。さらに取り込んでから再試行してください。
502Prediction failed.一時的なサーバー側の障害です。バックオフを挟んで再試行しても安全です。

完全なリファレンスについては、エラーとステータスコードを参照してください。
