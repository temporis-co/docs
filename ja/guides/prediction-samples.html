予測サンプルの扱い方

/v1/predict は多数のサンプリングされた軌道を返します。このガイドでは、それらを実際に必要なもの、すなわち点予測と信頼区間の帯に変換します。

なぜ多数の軌道なのか

Temporis はサンプリングによって予測を生成します。言語モデルがテキストを続けるのと同じように、履歴を読み取って、もっともらしい続きを描きます。各サンプルは未来そのものではなく、あり得る 1 つの未来です。count 個のサンプルを要求すると、predictions に count 個の独立した軌道が返されます。

多数持つことの意義はばらつきにあります。軌道が一致する箇所ではモデルは自信を持っており、扇状に広がる箇所では未来が本当に不確実です。そのばらつきこそがあなたの不確実性の推定であり、だからこそ単一のサンプルが望むものになることはまれです。

そのばらつきを解像できるよう count を十分高く設定してください。少数のサンプルではブレが大きく信頼できない描像になります。20 から 100 あたりであれば、ほとんどの系列で分布をうまく解像できます。多いほど滑らかになりますが、無料ではありません。

count はトークンを消費する

予測はモデルトークン単位で課金され、各軌道は予測範囲全体のトークンを生成します。count を 2 倍にすると、呼び出しのコストはおおよそ 2 倍になります。ばらつきがはっきり見える程度に十分なサンプルを使ったら、そこで止めてください。

点予測

サンプルを将来の各タイムスタンプにつき 1 つの数値にまとめるには、各タイムステップで軌道全体を集約します。中央値が良いデフォルトです。ときおり現れる極端な軌道に対して頑健だからです。お好みであれば平均値も使えます。

各予測の data 行は同じタイムスタンプを同じ順序で共有しているため、インデックスで揃えられます。ここでは hourly_orders プロファイルから 50 個のサンプルを要求し、タイムスタンプごとに orders の中央値予測を計算します。

import os, requests
from statistics import median

token = os.environ["TEMPORIS_TOKEN"]

resp = requests.post(
    "https://api.temporis.co/v1/predict",
    headers={"Authorization": f"Bearer {token}"},
    json={"data_profile": "hourly_orders", "count": 50, "temperature": 1.0, "top_p": 0.9},
)
resp.raise_for_status()
body = resp.json()

# Each prediction's rows are in the same timestamp order, so transpose by index.
preds = body["predictions"]            # length == count (50)
rows = list(zip(*[p["data"] for p in preds]))  # group row i across all samples

point_forecast = []
for step in rows:
    ts = step[0]["timestamp"]          # shared timestamp for this step
    values = [row["orders"] for row in step]
    point_forecast.append({"timestamp": ts, "orders": median(values)})

for f in point_forecast[:3]:
    print(f["timestamp"], round(f["orders"], 1))

信頼区間の帯

中央値は中心線を与え、帯はその周囲の範囲を与えます。サンプル全体にわたってタイムスタンプごとの分位点を計算してください。第 10、第 50、第 90 パーセンタイルがよく選ばれます。結果は 3 本の線、すなわち下端（p10）、中心（p50）、上端（p90）になります。

帯は「このタイムスタンプにおいて、サンプリングされた未来の約 80% が p10 と p90 の間に収まる」と読んでください。帯が狭ければモデルは自信を持っており、先を見るほど広がる帯は、遠い未来が近い未来より不確実であることをモデルが正直に伝えているのです。

import os, requests
import numpy as np

token = os.environ["TEMPORIS_TOKEN"]

resp = requests.post(
    "https://api.temporis.co/v1/predict",
    headers={"Authorization": f"Bearer {token}"},
    json={"data_profile": "hourly_orders", "count": 50, "temperature": 1.0, "top_p": 0.9},
)
resp.raise_for_status()
body = resp.json()

preds = body["predictions"]
timestamps = [row["timestamp"] for row in preds[0]["data"]]

# Shape: (count, horizon) — one row per sample, one column per timestep.
matrix = np.array([[row["orders"] for row in p["data"]] for p in preds])

p10, p50, p90 = np.percentile(matrix, [10, 50, 90], axis=0)

band = [
    {"timestamp": ts, "p10": lo, "p50": mid, "p90": hi}
    for ts, lo, mid, hi in zip(timestamps, p10, p50, p90)
]
for b in band[:3]:
    print(b["timestamp"], round(b["p10"], 1), round(b["p50"], 1), round(b["p90"], 1))

実践での count、temperature、top_p の選び方

サンプルを形作るつまみは 3 つあり、それぞれ異なる役割を担います。

  • count は解像度を制御します。ばらつきをどれだけ細かくサンプリングするかです。分布の形状は変えず、それがどれだけはっきり見えるかだけを変えます。
  • temperature（0 より大きい）はばらつきを制御します。値が高いほど多様で広範囲の未来を描き、低いほどモデルが最も確からしいと考える経路の近くにサンプルが集まります。1.0 前後が中立です。
  • top_p（0 から 1）は裾を切り詰めます。下げるとあり得ない極端な値をカットするので、驚くような飛びが減ります。1.0 は分布全体を保持します。

系列のブレやすさに応じた、妥当な出発点は次のとおりです。

系列の性格counttemperaturetop_p
安定して滑らか（一定の需要）20–300.80.9
標準的501.00.9
変動が激しくスパイクが多い（価格、バースト的なトラフィック）80–1001.10.95

これらは出発点として扱ってください。帯を描き、正直であるには狭すぎないか、あるいは役に立たないほど広すぎないかを見て、まず temperature を調整してください。

loss による順位付け

帯ではなく、代表的な単一の経路が欲しいこともあります。デモ、健全性チェック、あるいは「典型的な未来はどんな見た目か」を手早く確認したいときです。すべての軌道は loss を持っており、loss が低いほどモデルがその経路をより典型的だと判断したことを意味します。これでソートして、リストの先頭を取ってください。

# body is the parsed /v1/predict response
ranked = sorted(body["predictions"], key=lambda p: p["loss"])

most_typical = ranked[0]               # lowest loss = most representative path
for row in most_typical["data"][:3]:
    print(row["timestamp"], row["orders"])

# Or trim outliers: keep the most-typical 80% before computing a band.
keep = ranked[: int(len(ranked) * 0.8)]

実際の予測には、単一の軌道よりも中央値と帯を選んでください。loss が最も低い経路はあり得る 1 つの未来であって、それらの平均ではありません。順位付けは、きれいな例を選んだり外れ値を除外したりするのに最も役立ちます。
