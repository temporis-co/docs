クイックスタート

このガイドでは、新規アカウントから最初の予測までを案内します。API キーを作成し、 データソースを登録し、いくらかの履歴をストリーミングで送り込み、データプロファイルを 構築し、prediction エンドポイントを呼び出します。所要時間は約 5 分です。

2 つの面を持つ 1 つのプラットフォーム。 セットアップ作業 (キー、データソース、データプロファイル、課金) はダッシュボードで行い、データと予測の作業は API を通じて行います。このページでは両方を順を追って説明します。

  1. アクセストークンを作成する

    ダッシュボードで Access Tokens を開き、1 つ作成します。トークンは 一度だけ表示されるので、すぐにコピーしてください。ts_… のような 形式で、すべての API リクエストの認証に使われます。

    以下の例をそのまま動かせるように、環境変数として保存しておきましょう:

    export TEMPORIS_TOKEN="ts_your_token_here"

  2. 支払い方法を追加する

    予測は使用トークン量に応じて課金されるため、/v1/predict を実行する前に カードの登録が必要です。Billing から追加してください。 データの取り込みには課金は不要で、課金が必要なのは予測だけです。

  3. データソースを作成する

    Data Sources で store_metrics という名前のデータソースを 作成します。名前は小文字で始まり、小文字・数字・アンダースコアのみを含む必要があります。 データソースはデータを送信するまで空のコンテナです。

  4. 履歴を送信する

    各レコードは (timestamp, name, value) の 3 つ組です。 timestamp は秒単位の Unix 時刻、name は系列、value は数値です。ここでは 1 時間ごとの orders の値をいくつか送信しますが、実際には手元にある実履歴をできるだけ多く送信します。

  5. データプロファイルを構築する

    ダッシュボードに戻り、Data Profiles を開いて hourly_orders という名前のプロファイルを作成します。 store_metrics データソースを指定し、orders 系列を選択して、 interval を 3600 秒 (1 時間) に設定します。それ以外は すべてデフォルトのままで十分な出発点になります。

    Temporis がプロファイルを配信用に準備できるようになるには、ある程度の履歴のベースラインが 必要です。十分なデータが揃うと、ダッシュボードでそのプロファイルが ready と表示されます。それまでは予測が「not ready」エラーを返しますが、 これは想定どおりの動作です。

  6. 予測を生成する

    では、プロファイルに次に来るものを尋ねてみましょう。count は描画する独立した 未来の軌跡の数、temperature と top_p はサンプリングの大胆さを 制御します。

    "temperature": 1.0,

いま何が起きたのか

あなたは観測値をデータソースにストリーミングで送り込み、それをどう読み取るかを データプロファイルで記述し、/v1/predict から起こりうる 5 つの未来を 描画しました。各予測は完全な軌跡であり、それらが合わさって次に来るものの確率的な姿を形づくります。 予測の loss が低いほど、モデルはその軌跡をより典型的だと判断したことになります。

次のステップ

予測の仕組み →

単一の推測よりもサンプルを描画する方が優れている理由と、内部で起きていること。

データプロファイル →

interval、履歴の長さ、クリーニングを調整してデータに合わせる。

サンプルを扱う →

軌跡を点予測と信頼区間に変換する。
