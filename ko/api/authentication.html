인증

모든 Temporis API 요청은 베어러 토큰으로 인증합니다. 토큰을 생성하고, 전송하고, 안전하게 보관하는 방법을 소개합니다.

액세스 토큰

액세스 토큰은 대시보드의 Access Tokens에서 생성합니다. 토큰은 ts_ 접두사가 붙은 긴 랜덤 문자열입니다.

전체 토큰은 생성하는 순간에 한 번만 표시됩니다. Temporis는 서버 측에 토큰의 해시만 저장하며, 토큰 자체는 절대 저장하지 않습니다. 즉, 분실한 토큰은 복구할 수 없으므로 잃어버린 경우에는 삭제하고 새로 생성해야 합니다.

토큰을 비밀번호처럼 다루세요

토큰을 가진 사람은 누구나 데이터를 수집하고 예측을 실행할 수 있으며, 예측 비용은 여러분의 계정으로 청구됩니다. 그에 맞게 철저히 보호하세요.

요청 인증하기

모든 요청에서 Bearer 방식을 사용하여 Authorization 헤더에 토큰을 전송하세요:

Authorization: Bearer ts_your_token_here

이 문서 전반의 예제는 TEMPORIS_TOKEN 환경 변수에서 토큰을 읽어오므로, 비밀 정보가 소스 코드에 노출되지 않습니다. 셸에서 한 번 내보내 두세요:

export TEMPORIS_TOKEN="ts_your_token_here"

그런 다음 헤더로 첨부하세요:

curl https://api.temporis.co/v1/predict \
  -H "Authorization: Bearer $TEMPORIS_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{ "data_profile": "hourly_orders", "count": 5, "temperature": 1.0, "top_p": 0.9 }'
import os, requests

token = os.environ["TEMPORIS_TOKEN"]

resp = requests.post(
    "https://api.temporis.co/v1/predict",
    headers={"Authorization": f"Bearer {token}"},
    json={"data_profile": "hourly_orders", "count": 5, "temperature": 1.0, "top_p": 0.9},
)
resp.raise_for_status()
const token = process.env.TEMPORIS_TOKEN;

const resp = await fetch("https://api.temporis.co/v1/predict", {
  method: "POST",
  headers: {
    "Authorization": `Bearer ${token}`,
    "Content-Type": "application/json",
  },
  body: JSON.stringify({ data_profile: "hourly_orders", count: 5, temperature: 1.0, top_p: 0.9 }),
});

인증에 실패하는 경우

Authorization 헤더가 누락되었거나 형식이 잘못되었거나, 토큰을 알 수 없는 경우 API는 HTTP 401과 함께 무엇이 잘못되었는지 설명하는 JSON 본문으로 응답합니다:

{ "detail": "Invalid access token." }

헤더가 누락되었거나 형식이 잘못된 경우에는 대신 { "detail": "Missing access token." }을 반환합니다. 두 경우 모두 401입니다. 전체 응답 목록은 오류 및 상태 코드를 참고하세요.

토큰을 안전하게 보관하기

몇 가지 습관만으로도 토큰이 엉뚱한 사람의 손에 들어가는 것을 막을 수 있습니다:

  • 토큰을 절대 버전 관리에 커밋하지 마세요. 그리고 클라이언트 측 코드에도 절대 넣지 마세요. 브라우저와 모바일 앱은 소스 코드를 사용자에게 그대로 전달합니다.
  • 토큰은 환경 변수나 비밀 관리 도구에서 읽어오세요. 로그, 스크린샷, 티켓에는 토큰이 들어가지 않도록 하세요.
  • 서비스마다 별도의 토큰을 생성하세요. 그래야 다른 토큰에 영향을 주지 않고 하나만 폐기할 수 있습니다.
  • 대시보드에서 토큰을 삭제하면 즉시 폐기됩니다. 삭제는 곧바로 적용되며, 해당 토큰을 사용하는 다음 요청부터 401을 받게 됩니다.
