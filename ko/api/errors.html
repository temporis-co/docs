오류 및 상태 코드

Temporis가 성공과 실패를 보고하는 방식, 각 상태의 의미, 그리고 이를 처리하는 방법을 알아봅니다.

응답 형태

응답의 형태는 무슨 일이 일어났는지에 따라 달라집니다:

  • 성공한 수집은 본문 없이 204 No Content를 반환합니다.
  • 성공한 예측은 JSON 본문과 함께 200을 반환합니다.
  • 모든 오류는 해당 HTTP 상태와 함께 { "detail": "..." } 형식의 JSON 본문을 반환합니다.
{ "detail": "Data profile not found." }

오류 처리하기

대부분의 실패는 명확한 다음 조치로 이어집니다:

  • 401 — Authorization 헤더가 존재하는지, 토큰이 유효한지 확인하세요. 인증을 참고하세요.
  • 402 — 대시보드의 Billing에서 신용카드를 추가하거나 수정한 후 다시 시도하세요.
  • 404 — 데이터 소스 또는 데이터 프로파일이 존재하는지, 그리고 프로파일이 제공 준비가 되었는지 확인하세요.
  • predict의 409 — 프로파일이 최소 요구치를 충족할 때까지 이력을 더 수집한 후 다시 시도하세요.
  • ingest의 409 — 데이터 소스가 보관 처리되었습니다. 보관을 해제하거나 다른 소스에 기록하세요.
  • 502 — 일시적입니다. 지수 백오프로 재시도하세요.

상태 코드에 대한 간단한 분기 처리로 이 모든 경우를 다룰 수 있습니다:

import time, requests

def predict(payload, token, retries=3):
    for attempt in range(retries):
        resp = requests.post(
            "https://api.temporis.co/v1/predict",
            headers={"Authorization": f"Bearer {token}"},
            json=payload,
        )
        if resp.status_code == 200:
            return resp.json()
        if resp.status_code == 502:
            time.sleep(2 ** attempt)   # transient: back off and retry
            continue
        detail = resp.json().get("detail", "")
        if resp.status_code == 402:
            raise RuntimeError(f"Billing: {detail} — add a card in the dashboard.")
        if resp.status_code == 409:
            raise RuntimeError(f"Not ready: {detail} — ingest more history.")
        resp.raise_for_status()        # 400 / 401 / 404: fix the request
    raise RuntimeError("Prediction failed after retries.")

재시도

적절한 대상만 재시도하세요

502는 일시적이므로 재시도해도 안전합니다. 수집은 멱등하므로((timestamp, name) 키에 대해 업서트로 기록됨) 실패한 수집을 재시도하는 것은 언제나 안전합니다. 4xx 응답을 무턱대고 재시도하지 말고, 먼저 요청을 수정하세요.
