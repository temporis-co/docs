레코드 수집

데이터 소스로 관측값을 전송합니다. 데이터 소스는 이미 존재해야 하며(대시보드에서 생성), 보관 처리되지 않은 상태여야 합니다.

POST/v1/data_sources/ingest

요청 본문

본문에는 대상 데이터 소스의 이름과 레코드 배치가 담깁니다.

data_sourcestring필수

기록할 대상인, 보관 처리되지 않은 기존 데이터 소스의 이름입니다.

recordsarray필수

저장할 레코드 목록입니다. 각 레코드는 하나의 시계열을 특정 시점에 한 번 관측한 값입니다.

레코드 필드

timestampinteger필수

초 단위 Unix 시간입니다.

namestring필수

시계열 이름입니다. 하나의 데이터 소스는 여러 개의 이름이 붙은 시계열을 담습니다.

valuenumber필수

관측된 값입니다(정수 또는 부동소수점).

응답

성공 시 이 엔드포인트는 빈 본문과 함께 204 No Content를 반환합니다:

204 No Content

업서트 동작

각 레코드는 (timestamp, name) 쌍으로 키가 지정됩니다. 동일한 타임스탬프와 시계열 이름으로 레코드를 다시 보내면 저장된 값을 덮어씁니다.

재시도와 백필은 멱등합니다

쓰기가 업서트이기 때문에, 실패한 요청을 안전하게 재시도하거나 과거 데이터를 다시 보낼 수 있습니다. 중복된 항목은 누적되지 않고 동일한 키로 합쳐집니다.

배칭

요청당 한 개의 레코드를 보내는 것보다 한 번의 요청에 많은 레코드를 보내는 것이 좋습니다. 배칭은 왕복 횟수와 오버헤드를 크게 줄여 줍니다. 과거 데이터를 적재할 때는 한 번에 한 지점씩 스트리밍하지 말고, 많은 레코드를 묶은 청크 단위로 보내세요.

오류

오류는 HTTP 상태와 함께 { "detail": "..." } 형식의 JSON 본문을 반환합니다.

상태detail 메시지의미
401Invalid access token. / Missing access token.인증에 실패했습니다. 토큰을 확인하세요.
404Data source not found.지정한 데이터 소스가 존재하지 않습니다. 먼저 대시보드에서 생성하세요.
409Data source is archived.데이터 소스가 보관 처리되어 쓰기를 받을 수 없습니다. 보관을 해제하거나 다른 소스를 사용하세요.

전체 레퍼런스는 오류 및 상태 코드를 참고하세요.
