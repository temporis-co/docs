데이터 프로필

데이터 프로필은 Temporis에게 데이터 소스를 어떻게 읽을지 알려주는 레시피입니다. 어떤 시계열을, 어떤 주기로, 얼마만큼의 이력으로, 어떻게 정제할지를 정의합니다. 바로 이것이 여러분이 예측 대상으로 삼는 대상입니다.

프로필이 정의하는 것

데이터 소스는 원시 행을 담고 있습니다. 프로필은 이러한 행을 모델이 읽을 수 있는 깔끔하고 일정한 간격의 신호로 변환합니다 — 시계열을 선택하고, 관측값을 시간 격자에 맞춰 정렬하고, 얼마만큼의 이력을 사용할지 결정하고, 빈 구간을 정리합니다. predict를 호출할 때는 소스가 아니라 프로필을 지정합니다.

프로필은 대시보드에서 생성하고 편집합니다. 여기서의 예시는 hourly_orders입니다. 이 프로필은 store_metrics의 orders 시계열을 1시간 격자(interval 3600)로 읽습니다.

데이터 선택하기

소스와 시계열

프로필이 무엇을 읽을지는 두 가지 설정으로 선택합니다:

  • data_source — 읽어 들일 데이터 소스.
  • names — 해당 소스 내의 하나 이상의 시계열 이름.

이름을 하나만 지정하면 단변량(univariate) 예측이 됩니다. orders와 visitors처럼 여러 개를 지정하면 프로필이 다변량(multivariate)이 됩니다. 시계열이 공동으로 예측되므로 모델이 이들 간의 관계를 활용할 수 있습니다.

시간 격자

실제 관측값이 완벽하게 일정한 주기로 도착하는 경우는 드물기 때문에, 프로필은 이를 일정한 격자에 맞춰 리샘플링합니다.

interval은 각 격자 버킷의 크기를 초 단위로 나타냅니다. interval이 3600이면 관측값이 1시간 단위 버킷에 정렬됩니다. 격자는 여러분의 가장 최근 관측값을 기준으로 고정되어 거기서부터 뒤로 단계를 밟아가므로, 최신 데이터는 항상 격자에 깔끔하게 맞아떨어집니다.

모델은 모든 지점을 하나씩 읽지 않습니다. 대신 패치(patch) 단위, 즉 시계열의 고정 크기 청크 단위로 작동합니다:

  • patch_len — 하나의 패치를 구성하는 지점의 개수.
  • patch_spacing — 사용하는 지점 사이의 간격(stride)으로, 일종의 다운샘플링입니다.
  • num_patches — 컨텍스트로 사용할 이력 패치의 최대 개수.
  • min_patches — 예측을 제공하기 위해 필요한 이력 패치의 최소 개수.

직관적으로 말하면, 모델이 보는 컨텍스트는 대략 최근 이력 num_patches개 패치 분량입니다. 프로필에서 사용 가능한 패치가 min_patches개보다 적으면 예측을 제공할 수 없으며 HTTP 409 "Not enough data for this data profile."가 반환됩니다.

간격은 예측 해상도도 결정합니다. 예측 지점 사이의 실효 단계는 interval × patch_spacing초입니다. interval이 3600이고 patch_spacing이 1이면 예측값은 1시간 간격으로 나옵니다.

시계열 정제하기

리샘플링 후에는 일부 버킷에 관측값이 없을 수 있습니다. 이를 처리하는 방식은 두 가지 설정으로 결정됩니다:

  • fill_limit — 전방 채우기(forward-fill, 마지막으로 알려진 값을 앞으로 이어가는 것)가 적용되는 연속된 빈 버킷의 최대 길이입니다. 이보다 긴 간격은 실제로 누락된 것으로 간주되어 해당 행은 대신 제거됩니다.
  • exclude_window — 마스킹할 반복적인 하루 중 시간 구간으로, 시간대(timezone)와 시작 및 종료 시각으로 지정합니다. 시장이 닫힌 야간 시간대처럼 예상되는 비활동 시간에 사용하여, 그 조용한 구간이 시계열을 왜곡하지 않도록 합니다.

수준(level) 대 성장(growth)

어떤 시계열은 수준으로 모델링하는 것이 가장 좋고, 또 어떤 시계열은 상대적 변화로 모델링하는 것이 가장 좋습니다. use_diffs 설정(로그-차분 변환)은 모델이 원시 값 대신 로그 성장 — 한 단계에서 다음 단계로의 비례적 변화 — 을 기준으로 작동하도록 전환합니다.

곱셈적으로 움직이거나 넓은 범위에 걸쳐 강하게 추세를 보이는 시계열에는 이 옵션을 켜세요. 매출, 가격, 그리고 절대적 변화보다 10% 변화가 더 중요한 모든 경우가 해당됩니다. 유계이며 대략 정상(stationary)인 범위에 머물러 원시 수준 자체가 이미 예측하기에 자연스러운 시계열에는 이 옵션을 꺼두세요.

준비 상태

프로필은 생성하는 즉시 사용할 수 있는 것이 아닙니다. Temporis는 먼저 정규화 통계를 계산합니다 — 값을 모델이 기대하는 범위로 스케일링할 수 있도록 데이터의 전형적인 분포를 학습합니다. 이를 수행할 만큼 충분한 데이터가 모이기 전까지 프로필은 준비되지 않은 상태이며, 대시보드에 준비 상태가 표시됩니다.

준비되지 않은 프로필을 대상으로 예측하면 HTTP 404 "Data profile is not ready to serve."가 반환됩니다. 대시보드에서 프로필이 준비되었다고 표시될 때까지 기다린 후 predict를 호출하세요.
