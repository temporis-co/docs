예측과 샘플링

예측 요청은 하나가 아니라 여러 가지 가능한 미래를 반환합니다. 이 페이지에서는 응답을 읽는 방법과 샘플링 제어가 이를 어떻게 형성하는지 설명합니다. 정확한 스키마는 predict 레퍼런스를 참고하세요.

요청의 구조

예측은 원시 데이터 소스가 아니라 데이터 프로필을 대상으로 합니다. 프로필은 이미 어떤 시계열을 읽고 어떻게 인코딩할지 알고 있으므로, 요청은 어떤 프로필을 사용하고 어떻게 샘플링할지만 지정하면 됩니다. 네 개의 필드로 충분합니다:

data_profilestring필수

hourly_orders와 같은 여러분의 프로필 중 하나의 이름 — 또는 공개 프로필.

countinteger필수

추출할 독립적인 미래 궤적의 개수. 최소 1개입니다.

temperaturenumber필수

샘플이 얼마나 퍼져 있는지를 나타냅니다. 0보다 커야 하며, 약 1.0이 중립값입니다.

top_pnumber필수

0에서 1 범위의 뉴클리어스 샘플링 컷오프. 낮출수록 가능성이 낮은 꼬리 부분을 잘라냅니다.

다음은 hourly_orders 프로필을 대상으로 20개의 궤적을 요청하는 예시입니다:

curl https://api.temporis.co/v1/predict \
  -H "Authorization: Bearer ts_your_token" \
  -H "Content-Type: application/json" \
  -d '{
    "data_profile": "hourly_orders",
    "count": 20,
    "temperature": 1.0,
    "top_p": 0.9
  }'
import requests

resp = requests.post(
    "https://api.temporis.co/v1/predict",
    headers={"Authorization": "Bearer ts_your_token"},
    json={
        "data_profile": "hourly_orders",
        "count": 20,
        "temperature": 1.0,
        "top_p": 0.9,
    },
)
resp.raise_for_status()
result = resp.json()
const resp = await fetch("https://api.temporis.co/v1/predict", {
  method: "POST",
  headers: {
    "Authorization": "Bearer ts_your_token",
    "Content-Type": "application/json",
  },
  body: JSON.stringify({
    data_profile: "hourly_orders",
    count: 20,
    temperature: 1.0,
    top_p: 0.9,
  }),
});
const result = await resp.json();

모든 필드, 상태 코드, 오류 메시지를 포함한 전체 레퍼런스는 predict 엔드포인트 레퍼런스에 있습니다.

응답 읽기

호출이 성공하면 네 개의 최상위 필드와 함께 HTTP 200을 반환합니다:

request_idstring

이 요청에 대한 16진수 식별자. 지원 문의와 자체 로그를 위해 보관하세요.

data_profilestring

요청한 프로필을 그대로 돌려줍니다.

predict_frominteger

예측이 이어지는 마지막 관측된 타임스탬프의 Unix 초입니다. 모든 궤적은 이 지점 바로 다음부터 시작합니다.

predictionsarray

샘플링된 궤적마다 하나의 항목(길이는 count와 같음). 각 항목은 loss와 예측 행으로 이루어진 data 배열을 담습니다.

아래 예시는 orders 예측의 궤적 중 두 개를 보여줍니다. 프로필의 interval은 3600이고 patch_spacing은 2이므로, 행은 interval × patch_spacing = 7200초(2시간) 간격으로 진행됩니다:

{
  "request_id": "9f3c1a7e2b4d",
  "data_profile": "hourly_orders",
  "predict_from": 1718888400,
  "predictions": [
    {
      "loss": 0.84,
      "data": [
        { "timestamp": 1718895600, "orders": 142.0 },
        { "timestamp": 1718902800, "orders": 137.5 }
      ]
    },
    {
      "loss": 1.07,
      "data": [
        { "timestamp": 1718895600, "orders": 151.2 },
        { "timestamp": 1718902800, "orders": 129.8 }
      ]
    }
  ]
}

각 행에는 timestamp와 함께 프로필이 읽는 시계열마다 하나의 필드가 있습니다 — 여기서는 orders 하나뿐입니다. 타임스탬프는 interval × patch_spacing초 간격으로 진행되며, 이 간격은 모든 궤적에서 동일하므로 샘플을 행 단위로 정렬할 수 있습니다.

샘플링 제어

세 개의 파라미터가 돌려받는 미래의 집합을 형성합니다. 이들은 해상도와 다양성을 비용 및 안정성과 맞바꿉니다.

파라미터역할일반적인 값높이거나 낮춰야 할 때
count추출되는 독립 궤적의 개수. 샘플이 많을수록 분포를 더 잘 분해합니다.20에서 100더 매끄러운 구간을 원하면 높이고, 토큰 비용을 줄이려면 낮춥니다.
temperature샘플이 얼마나 퍼져 있는지. 높을수록 더 다양하고 불확실하며, 낮을수록 가장 가능성 높은 경로 근처에 모입니다.~1.0예측이 너무 좁아 보이면 높이고, 비현실적으로 흩어지면 낮춥니다.
top_p뉴클리어스 컷오프. 샘플링 전에 가능성이 낮은 꼬리 부분을 잘라냅니다.0.9에서 1.0예상치 못한 급변을 억제하려면 낮추고, 전체 분포를 유지하려면 1.0 쪽으로 높입니다.
비용은 count에 비례합니다

각 샘플은 독립적으로 생성되며 모델 사용량의 토큰당으로 과금되므로, count가 100인 요청은 count가 20인 요청보다 대략 다섯 배의 비용이 듭니다. 필요한 해상도와 비용 사이의 균형을 맞춰 count를 선택하세요.

Loss: 궤적 순위 매기기

모든 예측은 loss를 담고 있습니다. 값이 낮을수록 모델이 그 궤적을 더 전형적이라고 — 이력을 고려할 때 기대한 바에 더 부합한다고 — 판단했음을 의미합니다. loss가 높은 경로도 여전히 유효한 샘플이며, 단지 덜 중심적일 뿐입니다.

대표 궤적 하나를 고르거나 집합을 요약할 때의 가중치로 loss를 사용하세요. 가장 전형적인 경로를 가져오려면:

best = min(result["predictions"], key=lambda p: p["loss"])

다변량 예측

프로필이 여러 시계열 이름을 읽는 경우, 각 예측 행은 이름마다 하나의 필드를 담습니다 — 시계열이 공동으로 예측되므로 한 행 내에서 값들이 서로 일관성을 유지합니다. kw와 price를 함께 읽는 프로필은 다음과 같은 행을 반환합니다:

{ "timestamp": 1718895600, "kw": 482.0, "price": 31.7 }

공개 데이터 프로필

여러분이 따로 설정할 필요 없이 Temporis가 관리하는 공유 프로필을 대상으로 예측할 수 있습니다 — 공개 이름을 data_profile로 전달하기만 하면 됩니다:

{ "data_profile": "temporis/polymarket", "count": 20, "temperature": 1.0, "top_p": 0.9 }

응답 형태는 동일합니다. 각 행의 시계열 필드는 해당 공개 프로필이 읽는 것과 일치합니다.
