예측 샘플 다루기

/v1/predict는 수많은 샘플링된 궤적을 반환합니다. 이 가이드는 그것들을 실제로 필요한 것 — 점 예측과 신뢰 구간 — 으로 바꿉니다.

왜 여러 궤적인가

Temporis는 샘플링으로 예측을 생성합니다 — 이력을 읽고 그럴듯한 연속을 그려내는데, 마치 언어 모델이 텍스트를 이어가는 방식과 같습니다. 각 추출(draw)은 미래 그 자체가 아니라 가능한 하나의 미래입니다. count개의 추출을 요청하면 predictions 안에 count개의 독립적인 궤적을 받습니다.

여러 개를 갖는 핵심은 퍼짐(spread)입니다. 궤적들이 일치하는 곳에서는 모델이 확신하고, 부채꼴로 벌어지는 곳에서는 미래가 진정으로 불확실합니다. 그 퍼짐이 곧 불확실성 추정치이며 — 그래서 단일 샘플은 원하는 결과가 되기 어렵습니다.

그 퍼짐을 분해해 낼 만큼 count를 충분히 높게 설정하세요. 몇 개 안 되는 추출은 들쭉날쭉하고 신뢰할 수 없는 그림을 주며, 대부분의 시계열에서는 20에서 100 사이 정도면 분포를 잘 분해해 냅니다. 더 많으면 부드러워지지만 공짜는 아닙니다.

count는 토큰을 소모합니다

예측은 모델 토큰 단위로 과금되며, 모든 궤적은 전체 예측 구간만큼의 토큰을 생성합니다. count를 두 배로 늘리면 호출 비용도 대략 두 배가 됩니다. 퍼짐이 분명히 보일 만큼만 샘플을 사용하고 거기서 멈추세요.

점 예측

샘플들을 미래 타임스탬프마다 하나의 숫자로 압축하려면 각 타임스텝에서 궤적들을 가로질러 집계하세요. 중앙값(median)이 좋은 기본값입니다 — 가끔 나오는 극단적인 궤적에 강건하기 때문입니다. 원한다면 평균(mean)도 사용할 수 있습니다.

각 예측의 data 행은 같은 타임스탬프를 같은 순서로 공유하므로, 인덱스로 정렬할 수 있습니다. 다음은 hourly_orders 프로파일에서 샘플 50개를 요청하고 타임스탬프별 중앙값 orders 예측을 계산하는 예입니다:

import os, requests
from statistics import median

token = os.environ["TEMPORIS_TOKEN"]

resp = requests.post(
    "https://api.temporis.co/v1/predict",
    headers={"Authorization": f"Bearer {token}"},
    json={"data_profile": "hourly_orders", "count": 50, "temperature": 1.0, "top_p": 0.9},
)
resp.raise_for_status()
body = resp.json()

# Each prediction's rows are in the same timestamp order, so transpose by index.
preds = body["predictions"]            # length == count (50)
rows = list(zip(*[p["data"] for p in preds]))  # group row i across all samples

point_forecast = []
for step in rows:
    ts = step[0]["timestamp"]          # shared timestamp for this step
    values = [row["orders"] for row in step]
    point_forecast.append({"timestamp": ts, "orders": median(values)})

for f in point_forecast[:3]:
    print(f["timestamp"], round(f["orders"], 1))

신뢰 구간

중앙값은 중심선을 주고, 구간(band)은 그 주변의 범위를 줍니다. 샘플들을 가로질러 타임스탬프별 분위수를 계산하세요 — 10번째, 50번째, 90번째 백분위수가 흔한 선택입니다. 결과는 세 개의 선입니다: 하단 가장자리(p10), 중심(p50), 상단 가장자리(p90).

이 구간은 "이 타임스탬프에서 샘플링된 미래의 약 80%가 p10과 p90 사이에 들어온다"로 읽으세요. 좁은 구간은 모델이 확신한다는 뜻이고, 더 멀리 내다볼수록 넓어지는 구간은 먼 미래가 가까운 미래보다 덜 확실하다고 모델이 정직하게 알려주는 것입니다.

import os, requests
import numpy as np

token = os.environ["TEMPORIS_TOKEN"]

resp = requests.post(
    "https://api.temporis.co/v1/predict",
    headers={"Authorization": f"Bearer {token}"},
    json={"data_profile": "hourly_orders", "count": 50, "temperature": 1.0, "top_p": 0.9},
)
resp.raise_for_status()
body = resp.json()

preds = body["predictions"]
timestamps = [row["timestamp"] for row in preds[0]["data"]]

# Shape: (count, horizon) — one row per sample, one column per timestep.
matrix = np.array([[row["orders"] for row in p["data"]] for p in preds])

p10, p50, p90 = np.percentile(matrix, [10, 50, 90], axis=0)

band = [
    {"timestamp": ts, "p10": lo, "p50": mid, "p90": hi}
    for ts, lo, mid, hi in zip(timestamps, p10, p50, p90)
]
for b in band[:3]:
    print(b["timestamp"], round(b["p10"], 1), round(b["p50"], 1), round(b["p90"], 1))

실전에서 count, temperature, top_p 고르기

세 가지 손잡이가 샘플을 빚어내며, 각각 다른 일을 합니다:

  • count는 해상도를 제어합니다 — 퍼짐이 얼마나 세밀하게 샘플링되는지를 정합니다. 분포의 모양을 바꾸지는 않고, 그것을 얼마나 또렷이 보는지만 바꿉니다.
  • temperature(0보다 큼)는 퍼짐을 제어합니다 — 값이 높을수록 더 다양하고 폭넓은 미래를 그려내고, 낮을수록 모델이 가장 가능성 높다고 보는 경로 근처로 샘플을 모읍니다. 1.0 정도가 중립적입니다.
  • top_p(0에서 1)는 꼬리를 잘라냅니다 — 낮추면 가능성 낮은 극단을 잘라내 놀라운 점프가 줄어들고, 1.0은 분포 전체를 유지합니다.

시계열이 얼마나 들쭉날쭉한지에 따른 합리적인 출발점은 다음과 같습니다:

시계열 성격counttemperaturetop_p
안정적이고 매끄러움(꾸준한 수요)20–300.80.9
일반적501.00.9
변동이 크고 뾰족함(가격, 폭발적 트래픽)80–1001.10.95

이 값들은 출발점으로 삼으세요. 구간을 그려 보고, 정직하다고 보기엔 너무 빡빡한지 아니면 쓸모없을 만큼 너무 넓은지 살펴본 뒤 temperature부터 조정하세요.

loss로 순위 매기기

때로는 구간이 아니라 대표 경로 하나가 필요할 때가 있습니다 — 데모, 점검, 또는 "전형적인 미래는 어떤 모습일까"를 빠르게 보고 싶을 때요. 모든 궤적은 loss를 지니며, loss가 낮을수록 모델이 그 경로를 더 전형적으로 보았다는 뜻입니다. 이를 기준으로 정렬하고 목록의 앞쪽을 취하세요:

# body is the parsed /v1/predict response
ranked = sorted(body["predictions"], key=lambda p: p["loss"])

most_typical = ranked[0]               # lowest loss = most representative path
for row in most_typical["data"][:3]:
    print(row["timestamp"], row["orders"])

# Or trim outliers: keep the most-typical 80% before computing a band.
keep = ranked[: int(len(ranked) * 0.8)]

실제 예측을 위해서는 단일 궤적보다 중앙값과 구간을 선호하세요 — loss가 가장 낮은 경로는 가능한 하나의 미래이지 그것들의 평균이 아닙니다. 순위 매기기는 깔끔한 예시를 고르거나 이상치를 걸러낼 때 가장 유용합니다.
