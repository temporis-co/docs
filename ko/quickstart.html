빠른 시작

이 가이드는 새로 만든 계정에서 첫 예측까지 안내합니다. API 키를 생성하고, 데이터 소스를 등록하고, 약간의 이력을 스트리밍으로 보내고, 데이터 프로파일을 구축한 다음, 예측 엔드포인트를 호출하게 됩니다. 약 5분을 예상하세요.

두 개의 표면, 하나의 플랫폼. 설정은 대시보드에서(키, 데이터 소스, 데이터 프로파일, 결제) 하고, 데이터와 예측 작업은 API를 통해 합니다. 이 페이지는 둘 다를 안내합니다.

  1. 액세스 토큰 생성

    대시보드에서 Access Tokens를 열고 하나를 생성하세요. 토큰은 한 번만 표시됩니다 — 즉시 복사하세요. ts_…처럼 생겼으며 모든 API 요청을 인증합니다.

    아래 예제가 그대로 작동하도록 환경 변수로 저장하세요:

    export TEMPORIS_TOKEN="ts_your_token_here"

  2. 결제 수단 추가

    예측은 사용량의 토큰 단위로 과금되므로, /v1/predict를 실행하기 전에 등록된 카드가 필요합니다. Billing에서 하나를 추가하세요. 데이터 수집에는 결제가 필요하지 않습니다 — 예측에만 필요합니다.

  3. 데이터 소스 생성

    Data Sources에서 store_metrics라는 이름으로 하나를 생성하세요. 이름은 소문자로 시작해야 하며 소문자, 숫자, 밑줄만 포함할 수 있습니다. 데이터 소스는 데이터를 보내기 전까지는 빈 컨테이너입니다.

  4. 이력 보내기

    각 레코드는 (timestamp, name, value) 삼중쌍입니다. timestamp는 초 단위 Unix 시간이고, name은 시계열이며, value는 숫자입니다. 여기서는 시간별 orders 측정값 몇 개를 보내지만, 실제로는 가진 만큼 많은 실제 이력을 보내게 됩니다.

  5. 데이터 프로파일 구축

    다시 대시보드로 돌아가 Data Profiles를 열고 hourly_orders라는 이름으로 하나를 생성하세요. 이를 store_metrics 데이터 소스에 연결하고, orders 시계열을 선택한 다음, interval을 3600초(한 시간)로 설정하세요. 나머지 모든 것의 기본값은 시작점으로 손색이 없습니다.

    Temporis가 프로파일을 서빙용으로 준비할 수 있으려면 그 전에 기준이 되는 이력이 필요합니다. 충분한 데이터가 쌓이면 대시보드가 그것을 준비됨으로 표시합니다. 그때까지는 예측이 "준비되지 않음" 오류를 반환합니다 — 이는 예상된 동작입니다.

  6. 예측 생성

    이제 프로파일에 다음에 무엇이 올지 요청합니다. count는 몇 개의 독립적인 미래 궤적을 추출할지이고, temperature와 top_p는 샘플링이 얼마나 모험적일지를 제어합니다. 

    curl -X POST "https://api.temporis.co/v1/predict" \
  -H "Authorization: Bearer $TEMPORIS_TOKEN" \
  --json '{
    "data_profile": "hourly_orders",
    "count": 5,
    "temperature": 1.0,
    "top_p": 0.9
  }'

방금 무슨 일이 일어났나

여러분은 관측값을 데이터 소스로 스트리밍해 넣고, 그것을 어떻게 읽을지 데이터 프로파일로 설명한 다음, /v1/predict에서 다섯 개의 가능한 미래를 추출했습니다. 각 예측은 완전한 궤적이며, 함께 모이면 다음에 무엇이 올지에 대한 확률적 그림을 형성합니다. 예측의 loss가 낮을수록 모델이 그 궤적을 더 전형적이라고 본 것입니다.
