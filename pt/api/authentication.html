Autenticação

Cada requisição à API da Temporis se autentica com um token bearer. Veja como criar um, enviá-lo e mantê-lo seguro.

Tokens de acesso

Você cria tokens de acesso no painel, em Access Tokens. Um token é uma longa string aleatória com o prefixo ts_.

O token completo é exibido uma única vez, no momento em que você o cria. A Temporis armazena apenas um hash dele no servidor, nunca o token em si. Isso significa que um token perdido não pode ser recuperado — se você o perder, exclua-o e crie um novo.

Trate os tokens como senhas

Qualquer pessoa que tenha um dos seus tokens pode ingerir dados e executar previsões — e as previsões são cobradas na sua conta. Proteja-os adequadamente.

Autenticando uma requisição

Envie o token no cabeçalho Authorization em cada requisição, usando o esquema Bearer:

Authorization: Bearer ts_your_token_here

Os exemplos ao longo desta documentação leem o token de uma variável de ambiente TEMPORIS_TOKEN, de modo que o segredo nunca apareça no código-fonte. Exporte-a uma vez no seu shell:

export TEMPORIS_TOKEN="ts_your_token_here"

Em seguida, inclua-a como um cabeçalho:

curl https://api.temporis.co/v1/predict \
  -H "Authorization: Bearer $TEMPORIS_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{ "data_profile": "hourly_orders", "count": 5, "temperature": 1.0, "top_p": 0.9 }'
import os, requests

token = os.environ["TEMPORIS_TOKEN"]

resp = requests.post(
    "https://api.temporis.co/v1/predict",
    headers={"Authorization": f"Bearer {token}"},
    json={"data_profile": "hourly_orders", "count": 5, "temperature": 1.0, "top_p": 0.9},
)
resp.raise_for_status()
const token = process.env.TEMPORIS_TOKEN;

const resp = await fetch("https://api.temporis.co/v1/predict", {
  method: "POST",
  headers: {
    "Authorization": `Bearer ${token}`,
    "Content-Type": "application/json",
  },
  body: JSON.stringify({ data_profile: "hourly_orders", count: 5, temperature: 1.0, top_p: 0.9 }),
});

Quando a autenticação falha

Se o cabeçalho Authorization estiver ausente ou malformado, ou se o token for desconhecido, a API responde com HTTP 401 e um corpo JSON descrevendo o que deu errado:

{ "detail": "Invalid access token." }

Um cabeçalho ausente ou malformado retorna { "detail": "Missing access token." } em vez disso. Ambos são 401s. Consulte Erros e códigos de status para a lista completa de respostas.

Mantendo os tokens seguros

Alguns hábitos mantêm seus tokens longe das mãos erradas:

  • Nunca faça commit de tokens no controle de versão, e nunca os coloque em código do lado do cliente. Navegadores e aplicativos móveis distribuem seu código-fonte aos usuários.
  • Leia os tokens de variáveis de ambiente ou de um gerenciador de segredos. Mantenha-os fora de logs, capturas de tela e tíquetes.
  • Crie um token separado por serviço para poder revogar um sem afetar os outros.
  • Exclua um token no painel para revogá-lo instantaneamente. A exclusão entra em vigor imediatamente — a próxima requisição que usar esse token recebe um 401.
