Erros e códigos de status

Como a Temporis reporta sucesso e falha, o que cada status significa e como tratá-los.

Formato da resposta

O formato de uma resposta depende do que aconteceu:

  • Uma ingestão bem-sucedida retorna 204 No Content sem corpo.
  • Uma previsão bem-sucedida retorna 200 com um corpo JSON.
  • Qualquer erro retorna o status HTTP relevante com um corpo JSON no formato { "detail": "..." }.
{ "detail": "Data profile not found." }

404 Not FoundFonte de dados não encontrada (ingest); perfil de dados não encontrado ou não pronto para servir (predict).ingest, predict
409 ConflictFonte de dados arquivada (ingest); dados insuficientes (predict).ingest, predict
502 Bad GatewayA previsão falhou — um erro transitório no servidor.predict

Tratando erros

A maioria das falhas corresponde a uma ação seguinte clara:

  • 401 — verifique se o cabeçalho Authorization está presente e se o token é válido. Consulte Autenticação.
  • 402 — adicione ou corrija um cartão de crédito no painel, em Billing, e tente novamente.
  • 404 — certifique-se de que a fonte de dados ou o perfil de dados existe, e de que o perfil está pronto para servir.
  • 409 em predict — ingira mais histórico até o perfil atingir seu mínimo, então tente novamente.
  • 409 em ingest — a fonte de dados está arquivada; desarquive-a ou escreva em outra fonte.
  • 502 — transitório. Tente novamente com backoff exponencial.

Uma simples ramificação no código de status cobre todos esses casos:

import time, requests

def predict(payload, token, retries=3):
    for attempt in range(retries):
        resp = requests.post(
            "https://api.temporis.co/v1/predict",
            headers={"Authorization": f"Bearer {token}"},
            json=payload,
        )
        if resp.status_code == 200:
            return resp.json()
        if resp.status_code == 502:
            time.sleep(2 ** attempt)   # transient: back off and retry
            continue
        detail = resp.json().get("detail", "")
        if resp.status_code == 402:
            raise RuntimeError(f"Billing: {detail} — add a card in the dashboard.")
        if resp.status_code == 409:
            raise RuntimeError(f"Not ready: {detail} — ingest more history.")
        resp.raise_for_status()        # 400 / 401 / 404: fix the request
    raise RuntimeError("Prediction failed after retries.")

Retentativas

Repita as coisas certas

Um 502 é seguro de repetir — ele é transitório. A ingestão é idempotente (as escritas fazem upsert pela chave (timestamp, name)), então repetir uma ingestão que falhou é sempre seguro. Evite repetir cegamente respostas 4xx; corrija a requisição primeiro.
