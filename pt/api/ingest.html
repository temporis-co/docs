Ingerir registros

Envie observações para uma fonte de dados. A fonte de dados já deve existir (crie-a no painel) e não estar arquivada.

POST/v1/data_sources/ingest

Corpo da requisição

O corpo nomeia a fonte de dados de destino e carrega um lote de registros.

data_sourcestringObrigatório

O nome de uma fonte de dados existente e não arquivada na qual escrever.

recordsarrayObrigatório

Uma lista de registros a armazenar. Cada registro é uma única observação de uma série em um instante de tempo.

Campos do registro

timestampintegerObrigatório

Tempo Unix em segundos.

namestringObrigatório

O nome da série. Uma fonte de dados contém muitas séries nomeadas.

valuenumberObrigatório

O valor observado (um inteiro ou float).

Exemplo de requisição

Isto escreve quatro observações de orders na fonte de dados store_metrics, lendo o token do ambiente:

curl https://api.temporis.co/v1/data_sources/ingest \
  -H "Authorization: Bearer $TEMPORIS_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "data_source": "store_metrics",
    "records": [
      { "timestamp": 1718841600, "name": "orders", "value": 42 },
      { "timestamp": 1718845200, "name": "orders", "value": 51 },
      { "timestamp": 1718848800, "name": "orders", "value": 47 },
      { "timestamp": 1718852400, "name": "orders", "value": 63 }
    ]
  }'

Resposta

Em caso de sucesso, o endpoint retorna 204 No Content com um corpo vazio:

204 No Content

Comportamento de upsert

Cada registro é identificado pelo par (timestamp, name). Reenviar um registro com o mesmo timestamp e nome de série sobrescreve o valor armazenado.

Retentativas e backfills são idempotentes

Como a escrita é um upsert, você pode reenviar com segurança uma requisição que falhou ou reenviar dados históricos — duplicatas se consolidam na mesma chave em vez de se acumularem.

Agrupamento em lotes

Prefira enviar muitos registros em uma única requisição em vez de um registro por requisição. O agrupamento em lotes reduz drasticamente as idas e voltas e o overhead. Ao carregar histórico, envie-o em blocos de muitos registros cada, em vez de transmitir um ponto de cada vez.

Erros

Os erros retornam o status HTTP mais um corpo JSON no formato { "detail": "..." }.

Statusmensagem detailSignificado
401Invalid access token. / Missing access token.A autenticação falhou — verifique seu token.
404Data source not found.A fonte de dados nomeada não existe. Crie-a primeiro no painel.
409Data source is archived.A fonte de dados está arquivada e não pode aceitar escritas. Desarquive-a ou use outra fonte.

Consulte Erros e códigos de status para a referência completa.
