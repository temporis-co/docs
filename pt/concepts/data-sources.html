Fontes de dados

Uma fonte de dados é onde ficam suas observações brutas. Esta página aborda o modelo de dados do Temporis e como enviar dados para ele da forma correta.

Você cria uma fonte de dados uma vez no painel e, em seguida, transmite observações para ela pela API à medida que acontecem.

O modelo de dados

O Temporis armazena dados como linhas. Toda observação é uma única tripla: quando algo foi medido, o que foi medido e o número que você mediu.

timestamp
Tempo Unix em segundos (um inteiro). O momento em que a observação foi feita.
name
A série à qual este valor pertence — por exemplo orders ou revenue.
value
Um número. Valores em ponto flutuante são permitidos.

Uma única fonte de dados pode conter muitas séries nomeadas lado a lado. O campo name é o que as mantém distintas. Seus dados ficam isolados dos de qualquer outro cliente — uma fonte de dados pertence somente a você.

Criando uma fonte de dados

Você cria uma fonte de dados no painel, em Data Sources. Dê um nome a ela e estará pronta para receber dados.

Os nomes seguem algumas regras:

  • Começar com uma letra minúscula.
  • Usar apenas letras minúsculas, dígitos e underscores.
  • Até 48 caracteres.

Portanto, store_metrics e grid_load_eu são válidos; Store-Metrics e 2024data não são. Uma fonte de dados recém-criada fica vazia até você enviar observações para ela.

Enviando observações

Você adiciona dados com o endpoint de ingestão. Cada chamada carrega a data_source de destino e uma lista de records. Aqui enviamos algumas leituras de orders para store_metrics.

curl -X POST https://api.temporis.co/v1/data_sources/ingest \
  -H "Authorization: Bearer ts_<token>" \
  -H "Content-Type: application/json" \
  -d '{
    "data_source": "store_metrics",
    "records": [
      { "timestamp": 1718870400, "name": "orders", "value": 42 },
      { "timestamp": 1718874000, "name": "orders", "value": 51 },
      { "timestamp": 1718877600, "name": "orders", "value": 47 }
    ]
  }'
import requests

requests.post(
    "https://api.temporis.co/v1/data_sources/ingest",
    headers={"Authorization": "Bearer ts_<token>"},
    json={
        "data_source": "store_metrics",
        "records": [
            {"timestamp": 1718870400, "name": "orders", "value": 42},
            {"timestamp": 1718874000, "name": "orders", "value": 51},
            {"timestamp": 1718877600, "name": "orders", "value": 47},
        ],
    },
)  # -> 204 No Content
await fetch("https://api.temporis.co/v1/data_sources/ingest", {
  method: "POST",
  headers: {
    "Authorization": "Bearer ts_<token>",
    "Content-Type": "application/json",
  },
  body: JSON.stringify({
    data_source: "store_metrics",
    records: [
      { timestamp: 1718870400, name: "orders", value: 42 },
      { timestamp: 1718874000, name: "orders", value: 51 },
      { timestamp: 1718877600, name: "orders", value: 47 },
    ],
  }),
}); // -> 204 No Content

Uma ingestão bem-sucedida retorna HTTP 204 No Content com corpo vazio — não há nada para ler de volta, as linhas são simplesmente armazenadas. Consulte a referência de ingestão para os detalhes completos da requisição e dos erros.

Várias séries em uma fonte

Você não está limitado a uma série por fonte. Envie valores de name diferentes para a mesma data_source e eles coexistem como séries separadas. Uma loja pode acompanhar pedidos, receita e visitantes, todos em store_metrics.

{
  "data_source": "store_metrics",
  "records": [
    { "timestamp": 1718870400, "name": "orders",  "value": 42 },
    { "timestamp": 1718870400, "name": "revenue", "value": 1284.50 }
  ]
}

Manter séries relacionadas juntas é conveniente e permite prevê-las em conjunto mais tarde. Um perfil de dados é onde você escolhe quais desses nomes serão de fato lidos.

Atualizações e preenchimentos retroativos

A chave de upsert é o par (timestamp, name). Enviar um registro cujo timestamp e name já existem sobrescreve o valor armazenado em vez de adicionar uma duplicata.

Preenchimentos retroativos e novas tentativas são seguros

Como as escritas são idempotentes em (timestamp, name), você pode reenviar um lote após uma falha de rede, corrigir uma leitura errada ou carregar meses de histórico em qualquer ordem — o resultado é sempre o mesmo valor final por intervalo.

Arquivamento

Quando uma fonte não está mais coletando novos dados, você pode arquivá-la no painel. O arquivamento interrompe novas ingestões: uma requisição direcionada a uma fonte arquivada retorna HTTP 409 "Data source is archived."

O arquivamento não é destrutivo. Os dados existentes permanecem onde estão, e quaisquer perfis de dados construídos sobre a fonte continuam funcionando com os dados já armazenados.
