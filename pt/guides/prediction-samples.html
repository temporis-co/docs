Trabalhando com amostras de previsão

/v1/predict retorna muitas trajetórias amostradas. Este guia as transforma nas coisas de que você realmente precisa: uma previsão pontual e uma faixa de confiança.

Por que muitas trajetórias

O Temporis gera previsões por amostragem — ele lê seu histórico e traça uma continuação plausível, do jeito que um modelo de linguagem continua um texto. Cada traçado é um futuro possível, não o futuro. Peça count traçados e você obtém count trajetórias independentes em predictions.

O propósito de ter muitas é a dispersão. Onde as trajetórias concordam, o modelo está confiante; onde elas se abrem em leque, o futuro é genuinamente incerto. Essa dispersão é a sua estimativa de incerteza — é por isso que uma única amostra raramente é o que você quer.

Defina count alto o suficiente para resolver essa dispersão. Um punhado de traçados dá um quadro instável e não confiável; algo na faixa de 20 a 100 resolve bem a distribuição para a maioria das séries. Mais é mais suave, mas não é de graça.

Count custa tokens

A previsão é tarifada por token do modelo, e cada trajetória gera um horizonte completo de tokens. Dobrar count aproximadamente dobra o custo da chamada. Use amostras suficientes para ver a dispersão com clareza e então pare.

Uma previsão pontual

Para reduzir as amostras a um único número por timestamp futuro, agregue entre as trajetórias em cada passo de tempo. A mediana é um bom padrão — é robusta à eventual trajetória extrema; a média também funciona, se você preferir.

As linhas de data de cada previsão compartilham os mesmos timestamps na mesma ordem, então você pode alinhá-las por índice. Aqui solicitamos 50 amostras do perfil hourly_orders e calculamos uma previsão mediana de orders por timestamp:

import os, requests
from statistics import median

token = os.environ["TEMPORIS_TOKEN"]

resp = requests.post(
    "https://api.temporis.co/v1/predict",
    headers={"Authorization": f"Bearer {token}"},
    json={"data_profile": "hourly_orders", "count": 50, "temperature": 1.0, "top_p": 0.9},
)
resp.raise_for_status()
body = resp.json()

# Each prediction's rows are in the same timestamp order, so transpose by index.
preds = body["predictions"]            # length == count (50)
rows = list(zip(*[p["data"] for p in preds]))  # group row i across all samples

point_forecast = []
for step in rows:
    ts = step[0]["timestamp"]          # shared timestamp for this step
    values = [row["orders"] for row in step]
    point_forecast.append({"timestamp": ts, "orders": median(values)})

for f in point_forecast[:3]:
    print(f["timestamp"], round(f["orders"], 1))

Faixas de confiança

A mediana lhe dá uma linha central; uma faixa lhe dá o intervalo em torno dela. Calcule quantis por timestamp entre as amostras — os percentis 10, 50 e 90 são uma escolha comum. O resultado são três linhas: uma borda inferior (p10), o centro (p50) e uma borda superior (p90).

Leia a faixa como "cerca de 80% dos futuros amostrados caem entre p10 e p90 neste timestamp." Uma faixa estreita significa que o modelo está confiante; uma faixa que se alarga conforme você olha mais adiante é o modelo dizendo honestamente que o futuro distante é menos certo do que o próximo.

import os, requests
import numpy as np

token = os.environ["TEMPORIS_TOKEN"]

resp = requests.post(
    "https://api.temporis.co/v1/predict",
    headers={"Authorization": f"Bearer {token}"},
    json={"data_profile": "hourly_orders", "count": 50, "temperature": 1.0, "top_p": 0.9},
)
resp.raise_for_status()
body = resp.json()

preds = body["predictions"]
timestamps = [row["timestamp"] for row in preds[0]["data"]]

# Shape: (count, horizon) — one row per sample, one column per timestep.
matrix = np.array([[row["orders"] for row in p["data"]] for p in preds])

p10, p50, p90 = np.percentile(matrix, [10, 50, 90], axis=0)

band = [
    {"timestamp": ts, "p10": lo, "p50": mid, "p90": hi}
    for ts, lo, mid, hi in zip(timestamps, p10, p50, p90)
]
for b in band[:3]:
    print(b["timestamp"], round(b["p10"], 1), round(b["p50"], 1), round(b["p90"], 1))

Escolhendo count, temperature, top_p na prática

Três botões moldam as amostras, e eles fazem trabalhos diferentes:

  • count controla a resolução — quão finamente a dispersão é amostrada. Ele não muda a forma da distribuição, apenas o quão claramente você a vê.
  • temperature (maior que 0) controla a dispersão — valores mais altos traçam futuros mais diversos e de maior amplitude; valores mais baixos agrupam as amostras perto do caminho mais provável do modelo. Cerca de 1.0 é neutro.
  • top_p (0 a 1) apara as caudas — diminuí-lo corta os extremos improváveis, então você obtém menos saltos surpreendentes; 1.0 mantém a distribuição completa.

Pontos de partida sensatos, dependendo de quão instável é a sua série:

Caráter da sériecounttemperaturetop_p
Estável, suave (demanda constante)20–300.80.9
Típica501.00.9
Volátil, com picos (preços, tráfego em rajadas)80–1001.10.95

Trate esses valores como um ponto de partida. Trace uma faixa, observe se ela está apertada demais para ser honesta ou tão ampla que é inútil, e ajuste o temperature primeiro.

Classificando por loss

Às vezes você quer um único caminho representativo em vez de uma faixa — para uma demonstração, uma verificação de sanidade ou um rápido "como é um futuro típico". Cada trajetória carrega um loss, e um loss menor significa que o modelo considerou aquele caminho mais típico. Ordene por ele e pegue o início da lista:

# body is the parsed /v1/predict response
ranked = sorted(body["predictions"], key=lambda p: p["loss"])

most_typical = ranked[0]               # lowest loss = most representative path
for row in most_typical["data"][:3]:
    print(row["timestamp"], row["orders"])

# Or trim outliers: keep the most-typical 80% before computing a band.
keep = ranked[: int(len(ranked) * 0.8)]

Para uma previsão de verdade, prefira a mediana e uma faixa a qualquer trajetória única — o caminho de menor loss é um futuro plausível, não uma média deles. A classificação é mais útil para escolher um exemplo limpo ou filtrar valores atípicos.
