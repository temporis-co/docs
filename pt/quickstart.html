Início rápido

Este guia leva você de uma conta recém-criada à sua primeira previsão. Você criará uma chave de API, registrará uma fonte de dados, enviará algum histórico, construirá um perfil de dados e chamará o endpoint de previsão. Reserve cerca de cinco minutos.

Duas superfícies, uma plataforma. Você configurará as coisas no painel (chaves, fontes de dados, perfis de dados, faturamento) e fará o trabalho de dados e previsão pela API. Esta página percorre ambos.

  1. Crie um token de acesso

    No painel, abra Tokens de Acesso e crie um. O token é exibido uma única vez — copie-o imediatamente. Ele se parece com ts_… e autentica cada requisição à API.

    Armazene-o como uma variável de ambiente para que os exemplos abaixo funcionem como estão:

    export TEMPORIS_TOKEN="ts_your_token_here"

  2. Adicione um método de pagamento

    A previsão é tarifada por token de uso, então um cartão cadastrado é obrigatório antes que /v1/predict seja executado. Adicione um em Faturamento. A ingestão de dados não exige faturamento — apenas a previsão exige.

  3. Crie uma fonte de dados

    Em Fontes de Dados, crie uma chamada store_metrics. Os nomes devem começar com uma letra minúscula e conter apenas letras minúsculas, dígitos e sublinhados. Uma fonte de dados é um contêiner vazio até você enviar dados a ela.

  4. Envie algum histórico

    Cada registro é uma tripla (timestamp, name, value). O timestamp é um tempo Unix em segundos, name é a série e value é o número. Aqui enviamos algumas leituras horárias de orders; na prática, você enviaria todo o histórico real que tiver.

    Uma ingestão bem-sucedida retorna 204 No Content — um corpo vazio. Enviar o mesmo (timestamp, name) novamente simplesmente sobrescreve o valor, então repetições e preenchimentos retroativos são seguros. Consulte Ingerir registros para os detalhes.

  5. Construa um perfil de dados

    De volta ao painel, abra Perfis de Dados e crie um chamado hourly_orders. Aponte-o para a fonte de dados store_metrics, selecione a série orders e defina o intervalo como 3600 segundos (uma hora). Os valores padrão para todo o resto são um bom ponto de partida.

    Um perfil precisa de uma linha de base de histórico antes que o Temporis possa prepará-lo para servir. Assim que tiver dados suficientes, o painel o marca como pronto. Até lá, as previsões retornam um erro de "não pronto" — isso é esperado.

  6. Gere uma previsão

    Agora peça ao perfil o que vem a seguir. count é quantas trajetórias futuras independentes desenhar; temperature e top_p controlam o quão ousada é a amostragem.

  -H "Authorization: Bearer $TEMPORIS_TOKEN" \
  --json '{
    "data_profile": "hourly_orders",
    "count": 5,
    "temperature": 1.0,
    "top_p": 0.9
  }'

O que acabou de acontecer

Você enviou observações para uma fonte de dados, descreveu como lê-las com um perfil de dados e desenhou cinco futuros possíveis a partir de /v1/predict. Cada previsão é uma trajetória completa; juntas, elas formam um quadro probabilístico do que vem a seguir. Quanto menor o loss de uma previsão, mais típica o modelo considerou essa trajetória.
