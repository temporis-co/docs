Ошибки и коды статусов

Как Temporis сообщает об успехе и неудаче, что означает каждый статус и как их обрабатывать.

Форма ответа

Форма ответа зависит от того, что произошло:

  • Успешная загрузка возвращает 204 No Content без тела.
  • Успешный прогноз возвращает 200 с JSON-телом.
  • Любая ошибка возвращает соответствующий HTTP-статус с JSON-телом вида { "detail": "..." }.
{ "detail": "Data profile not found." }

Коды статусов

СтатусЗначениеГде
200 OKПрогноз выполнен успешно; тело содержит выборки.predict
204 No ContentЗагрузка выполнена успешно; тела нет.ingest
400 Bad RequestНедопустимые параметры прогноза.predict
401 UnauthorizedОтсутствует или неверный токен доступа.все запросы
402 Payment RequiredНет привязанной карты или проблема с оплатой.predict
404 Not FoundИсточник данных не найден (ingest); профиль данных не найден или не готов к работе (predict).ingest, predict
409 ConflictИсточник данных архивирован (ingest); недостаточно данных (predict).ingest, predict
502 Bad GatewayПрогноз не выполнен — временная ошибка на стороне сервера.predict

Обработка ошибок

Большинство сбоев соответствуют понятному следующему действию:

  • 401 — убедитесь, что заголовок Authorization присутствует и токен действителен. См. Аутентификация.
  • 402 — добавьте или исправьте кредитную карту в панели управления в разделе «Биллинг», затем повторите попытку.
  • 404 — убедитесь, что источник данных или профиль данных существует и что профиль готов к работе.
  • 409 при predict — загружайте больше истории, пока профиль не достигнет своего минимума, затем повторите попытку.
  • 409 при ingest — источник данных архивирован; разархивируйте его или пишите в другой источник.
  • 502 — временная ошибка. Повторите с экспоненциальной задержкой.

Простое ветвление по коду статуса покрывает все эти случаи:

import time, requests

def predict(payload, token, retries=3):
    for attempt in range(retries):
        resp = requests.post(
            "https://api.temporis.co/v1/predict",
            headers={"Authorization": f"Bearer {token}"},
            json=payload,
        )
        if resp.status_code == 200:
            return resp.json()
        if resp.status_code == 502:
            time.sleep(2 ** attempt)   # transient: back off and retry
            continue
        detail = resp.json().get("detail", "")
        if resp.status_code == 402:
            raise RuntimeError(f"Billing: {detail} — add a card in the dashboard.")
        if resp.status_code == 409:
            raise RuntimeError(f"Not ready: {detail} — ingest more history.")
        resp.raise_for_status()        # 400 / 401 / 404: fix the request
    raise RuntimeError("Prediction failed after retries.")

Повторные попытки

Повторяйте то, что нужно

502 безопасно повторять — это временная ошибка. Загрузка идемпотентна (записи обновляются по ключу (timestamp, name)), поэтому повторять неудавшуюся загрузку всегда безопасно. Избегайте слепого повтора ответов 4xx; сначала исправьте запрос.
