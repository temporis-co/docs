Загрузка записей

Отправляйте наблюдения в источник данных. Источник данных уже должен существовать (создайте его в панели управления) и не быть архивированным.

POST/v1/data_sources/ingest

Тело запроса

Тело указывает целевой источник данных и содержит пакет записей.

data_sourcestringОбязательный

Имя существующего, неархивированного источника данных, в который выполняется запись.

recordsarrayОбязательный

Список записей для сохранения. Каждая запись — это одно наблюдение одного ряда в один момент времени.

Поля записи

timestampintegerОбязательный

Unix-время в секундах.

namestringОбязательный

Имя ряда. Один источник данных содержит множество именованных рядов.

valuenumberОбязательный

Наблюдаемое значение (целое число или число с плавающей точкой).

Пример запроса

Этот запрос записывает четыре наблюдения orders в источник данных store_metrics, читая токен из окружения:

curl https://api.temporis.co/v1/data_sources/ingest \
  -H "Authorization: Bearer $TEMPORIS_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "data_source": "store_metrics",
    "records": [
      { "timestamp": 1718841600, "name": "orders", "value": 42 },
      { "timestamp": 1718845200, "name": "orders", "value": 51 },
      { "timestamp": 1718848800, "name": "orders", "value": 47 },
      { "timestamp": 1718852400, "name": "orders", "value": 63 }
    ]
  }'

Ответ

В случае успеха эндпоинт возвращает 204 No Content с пустым телом:

204 No Content

Поведение upsert

Каждая запись идентифицируется парой (timestamp, name). Повторная отправка записи с той же меткой времени и именем ряда перезаписывает сохранённое значение.

Повторы и дозагрузки идемпотентны

Поскольку запись выполняется как upsert, вы можете безопасно повторить неудавшийся запрос или повторно отправить исторические данные — дубликаты схлопываются на один ключ, а не накапливаются.

Пакетная отправка

Предпочтительнее отправлять много записей в одном запросе, чем по одной записи на запрос. Пакетная отправка резко сокращает число обращений и накладные расходы. При загрузке истории отправляйте её порциями по множеству записей, а не передавайте по одной точке за раз.

Ошибки

Ошибки возвращают HTTP-статус и JSON-тело вида { "detail": "..." }.

СтатусСообщение detailЗначение
401Invalid access token. / Missing access token.Аутентификация не удалась — проверьте свой токен.
404Data source not found.Указанный источник данных не существует. Сначала создайте его в панели управления.
409Data source is archived.Источник данных архивирован и не может принимать записи. Разархивируйте его или используйте другой источник.

Полный справочник см. в разделе Ошибки и коды состояния.
