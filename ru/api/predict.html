Создание прогноза

Получите одно или несколько сэмплированных вариантов будущего из профиля данных. Требуется привязанная карта.

POST/v1/predict

Тело запроса

Вы указываете профиль данных и форму нужной вам выборки.

data_profilestringОбязательный

Имя одного из ваших профилей или публичного профиля, например temporis/polymarket.

countintegerОбязательный

Количество независимых будущих траекторий для получения. Должно быть не меньше 1.

temperaturenumberОбязательный

Температура сэмплирования, больше 0. Чем выше значение, тем шире разброс выборок; чем ниже — тем ближе они к наиболее вероятному пути.

top_pnumberОбязательный

Порог nucleus-сэмплирования в диапазоне от 0 до 1. Чем ниже значение, тем сильнее отсекается маловероятный хвост; 1.0 сохраняет полное распределение.

count, temperature и top_p подробно описаны в разделе Прогнозы.

Пример запроса

Этот запрос получает пять траекторий из профиля hourly_orders при нейтральной температуре:

curl https://api.temporis.co/v1/predict \
  -H "Authorization: Bearer $TEMPORIS_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "data_profile": "hourly_orders",
    "count": 5,
    "temperature": 1.0,
    "top_p": 0.9
  }'

Поля ответа

Успешный запрос возвращает HTTP 200 с таким телом.

request_idstring

Уникальный идентификатор этого запроса на прогноз.

data_profilestring

Повторяет имя профиля, которое вы запросили.

predict_frominteger

Unix-секунды: последняя наблюдённая метка времени, от которой продолжается прогноз. Каждая траектория шагает вперёд отсюда.

predictionsarray

Сэмплированные траектории. Длина равна запрошенному вами значению count.

Объект прогноза

lossnumber

Чем меньше значение, тем более типичной модель считает эту траекторию. Используйте его, чтобы ранжировать выборки или выбрать репрезентативную.

dataarray

Строки прогноза. В каждой строке есть timestamp плюс одно поле на каждое имя ряда в профиле.

Публичные профили данных

Вы можете прогнозировать общий датасет без какой-либо настройки с вашей стороны, передав имя публичного профиля в data_profile. Они поддерживаются Temporis и готовы к работе:

{ "data_profile": "temporis/polymarket", "count": 3, "temperature": 1.0, "top_p": 0.9 }

Ошибки

Ошибки возвращают HTTP-статус и JSON-тело вида { "detail": "..." }.

СтатусСообщение detailЗначение
400Invalid prediction parameters.Параметр вне допустимого диапазона, например count меньше 1 или temperature не больше 0.
402Add a credit card before using the API. / There is a payment issue. Add a new credit card.Нет активного биллинга. Добавьте или исправьте карту в панели управления.
404Data profile not found.Профиля с таким именем не существует.
404Data profile is not ready to serve.Профиль существует, но ещё не завершил подготовку.
409Not enough data for this data profile.У профиля меньше истории, чем требуется минимально. Загрузите больше данных и повторите попытку.
502Prediction failed.Временный сбой на стороне сервера. Безопасно повторить с задержкой.

Полный справочник см. в разделе Ошибки и коды статусов.
