Профили данных

Профиль данных — это рецепт, который указывает Temporis, как читать источник данных: какие ряды, с какой частотой, сколько истории и как её очищать. Именно по нему вы строите прогноз.

Что определяет профиль

Источник данных хранит необработанные строки. Профиль превращает эти строки в чистый, равномерно распределённый сигнал, который может читать модель: выбирает ряды, привязывает наблюдения к временной сетке, решает, сколько истории использовать, и устраняет пропуски. Когда вы вызываете прогноз, вы указываете профиль, а не источник.

Профили создаются и редактируются на панели управления. В примере здесь используется hourly_orders: он читает ряд orders из store_metrics на часовой сетке (interval 3600).

Выбор данных

Источник и ряды

Два параметра выбирают, что читает профиль:

  • data_source — из какого источника данных читать.
  • names — одно или несколько имён рядов внутри этого источника.

Указание одного имени даёт одномерный прогноз. Указание нескольких — скажем, orders и visitors — делает профиль многомерным: ряды прогнозируются совместно, так что модель может использовать связь между ними.

Временная сетка

Реальные наблюдения редко поступают с идеально равномерной частотой, поэтому профиль пересчитывает их на регулярную сетку.

interval — это размер каждой ячейки сетки в секундах. При interval 3600 наблюдения привязываются к часовым ячейкам. Сетка закрепляется на вашем самом последнем наблюдении и отсчитывается назад от него, так что свежие данные всегда чисто ложатся на сетку.

Модель не читает каждую точку по одной. Она работает с патчами — фрагментами ряда фиксированного размера:

  • patch_len — сколько точек составляют один патч.
  • patch_spacing — шаг между используемыми точками, форма прореживания.
  • num_patches — максимальное число патчей истории, используемых в качестве контекста.
  • min_patches — минимальное число патчей истории, необходимое для выдачи прогноза.

Интуитивно: контекст, который видит модель, — это примерно num_patches патчей недавней истории. Если у профиля доступно меньше min_patches патчей, он не может выдать результат, и вы получаете HTTP 409 "Not enough data for this data profile."

Шаг также задаёт разрешение вашего прогноза. Эффективный шаг между точками прогноза составляет interval × patch_spacing секунд. При interval 3600 и patch_spacing 1 прогнозы располагаются с интервалом в один час.

Очистка ряда

После пересчёта в некоторых ячейках может не оказаться наблюдений. Два параметра определяют, как это обрабатывается:

  • fill_limit — самая длинная серия подряд идущих пустых ячеек, которая будет заполнена вперёд (с переносом последнего известного значения). Пропуск длиннее этого считается действительно отсутствующим, и такие строки вместо этого отбрасываются.
  • exclude_window — повторяющееся окно времени суток, которое нужно маскировать, задаётся как часовой пояс плюс начало и конец. Используйте его для ожидаемого мёртвого времени, например ночных часов, когда рынок закрыт, чтобы этот спокойный промежуток не искажал ряд.

Уровни и рост

Некоторые ряды лучше всего моделировать по их уровню; другие — по их относительному изменению. Параметр use_diffs (преобразование лог-разностей) переключает модель на работу с логарифмическим ростом — пропорциональным изменением от шага к шагу — вместо необработанного значения.

Включайте его для рядов, которые движутся мультипликативно или сильно трендят в широком диапазоне: выручка, цены, всё, где изменение на 10% важнее абсолютного. Оставляйте его выключенным для рядов, которые находятся в ограниченном, примерно стационарном диапазоне, где необработанный уровень уже является естественной величиной для прогноза.

Готовность

Профиль не готов к использованию сразу после создания. Сначала Temporis вычисляет статистики нормализации — изучает типичный разброс ваших данных, чтобы масштабировать значения в диапазон, ожидаемый моделью. Пока данных для этого недостаточно, профиль не готов, и панель управления показывает его состояние готовности.

Прогноз по профилю, который не готов, возвращает HTTP 404 "Data profile is not ready to serve." Дождитесь, пока панель управления сообщит, что профиль готов, затем вызывайте прогноз.
