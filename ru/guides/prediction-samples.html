Работа с выборками прогнозов

/v1/predict возвращает множество выборочных траекторий. Это руководство превращает их в то, что вам действительно нужно: точечный прогноз и доверительный диапазон.

Зачем нужно много траекторий

Temporis формирует прогнозы путём сэмплирования — он читает вашу историю и рисует правдоподобное продолжение, подобно тому как языковая модель продолжает текст. Каждая выборка — это одно возможное будущее, а не само будущее. Запросите count выборок, и вы получите count независимых траекторий в predictions.

Смысл их множества — в разбросе. Там, где траектории согласуются, модель уверена; там, где они расходятся веером, будущее по-настоящему неопределённо. Этот разброс и есть ваша оценка неопределённости — поэтому одна выборка редко то, что вам нужно.

Установите count достаточно высоким, чтобы разрешить этот разброс. Горстка выборок даёт дёрганую, ненадёжную картину; значение где-то в диапазоне от 20 до 100 хорошо разрешает распределение для большинства рядов. Больше — глаже, но не бесплатно.

Count стоит токенов

Прогнозирование тарифицируется по токенам модели, и каждая траектория генерирует полный горизонт токенов. Удвоение count примерно удваивает стоимость вызова. Используйте столько выборок, сколько нужно, чтобы ясно увидеть разброс, и остановитесь.

Точечный прогноз

Чтобы свернуть выборки в одно число на каждую будущую временную метку, агрегируйте по траекториям на каждом шаге времени. Медиана — хороший вариант по умолчанию: она устойчива к отдельной шальной траектории; среднее тоже подойдёт, если вам так больше нравится.

Строки data каждого прогноза содержат одни и те же временные метки в одном и том же порядке, поэтому их можно сопоставить по индексу. Здесь мы запрашиваем 50 выборок из профиля hourly_orders и вычисляем медианный прогноз orders на каждую временную метку:

import os, requests
from statistics import median

token = os.environ["TEMPORIS_TOKEN"]

resp = requests.post(
    "https://api.temporis.co/v1/predict",
    headers={"Authorization": f"Bearer {token}"},
    json={"data_profile": "hourly_orders", "count": 50, "temperature": 1.0, "top_p": 0.9},
)
resp.raise_for_status()
body = resp.json()

# Each prediction's rows are in the same timestamp order, so transpose by index.
preds = body["predictions"]            # length == count (50)
rows = list(zip(*[p["data"] for p in preds]))  # group row i across all samples

point_forecast = []
for step in rows:
    ts = step[0]["timestamp"]          # shared timestamp for this step
    values = [row["orders"] for row in step]
    point_forecast.append({"timestamp": ts, "orders": median(values)})

for f in point_forecast[:3]:
    print(f["timestamp"], round(f["orders"], 1))

Доверительные диапазоны

Медиана даёт вам центральную линию; диапазон даёт область вокруг неё. Вычислите квантили по каждой временной метке по выборкам — 10-й, 50-й и 90-й перцентили — распространённый выбор. В результате получаются три линии: нижняя граница (p10), центр (p50) и верхняя граница (p90).

Читайте диапазон как «около 80% выборочных будущих попадают между p10 и p90 на этой временной метке». Узкий диапазон означает, что модель уверена; диапазон, расширяющийся по мере того, как вы смотрите дальше вперёд, — это модель честно сообщает вам, что далёкое будущее менее определённо, чем ближнее.

import os, requests
import numpy as np

token = os.environ["TEMPORIS_TOKEN"]

resp = requests.post(
    "https://api.temporis.co/v1/predict",
    headers={"Authorization": f"Bearer {token}"},
    json={"data_profile": "hourly_orders", "count": 50, "temperature": 1.0, "top_p": 0.9},
)
resp.raise_for_status()
body = resp.json()

preds = body["predictions"]
timestamps = [row["timestamp"] for row in preds[0]["data"]]

# Shape: (count, horizon) — one row per sample, one column per timestep.
matrix = np.array([[row["orders"] for row in p["data"]] for p in preds])

p10, p50, p90 = np.percentile(matrix, [10, 50, 90], axis=0)

band = [
    {"timestamp": ts, "p10": lo, "p50": mid, "p90": hi}
    for ts, lo, mid, hi in zip(timestamps, p10, p50, p90)
]
for b in band[:3]:
    print(b["timestamp"], round(b["p10"], 1), round(b["p50"], 1), round(b["p90"], 1))

Выбор count, temperature, top_p на практике

Тремя ручками задаётся вид выборок, и они выполняют разные задачи:

  • count управляет разрешением — насколько мелко сэмплируется разброс. Он не меняет форму распределения, лишь то, насколько ясно вы его видите.
  • temperature (больше 0) управляет разбросом — более высокие значения рисуют более разнообразные, шире разбросанные будущие; более низкие группируют выборки вблизи наиболее вероятного пути модели. Около 1.0 — нейтрально.
  • top_p (от 0 до 1) обрезает хвосты — снижение его отсекает маловероятные крайности, так что вы получаете меньше неожиданных скачков; 1.0 сохраняет полное распределение.

Разумные отправные точки, в зависимости от того, насколько дёрганый ваш ряд:

Характер рядаcounttemperaturetop_p
Стабильный, плавный (ровный спрос)20–300.80.9
Типичный501.00.9
Волатильный, скачкообразный (цены, всплесковый трафик)80–1001.10.95

Воспринимайте это как точку отсчёта. Постройте диапазон, посмотрите, не слишком ли он узок, чтобы быть честным, или настолько широк, что бесполезен, и сначала подстройте temperature.

Ранжирование по loss

Иногда вам нужен один представительный путь, а не диапазон — для демонстрации, проверки на здравый смысл или быстрого «как выглядит типичное будущее». Каждая траектория несёт значение loss, и меньший loss означает, что модель сочла этот путь более типичным. Отсортируйте по нему и возьмите начало списка:

# body is the parsed /v1/predict response
ranked = sorted(body["predictions"], key=lambda p: p["loss"])

most_typical = ranked[0]               # lowest loss = most representative path
for row in most_typical["data"][:3]:
    print(row["timestamp"], row["orders"])

# Or trim outliers: keep the most-typical 80% before computing a band.
keep = ranked[: int(len(ranked) * 0.8)]

Для настоящего прогноза предпочитайте медиану и диапазон любой отдельной траектории — путь с наименьшим loss это одно правдоподобное будущее, а не их среднее. Ранжирование наиболее полезно для выбора чистого примера или отсеивания выбросов.
