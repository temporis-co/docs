身份验证

每个 Temporis API 请求都使用 bearer 令牌进行身份验证。下面介绍如何创建、发送并妥善保管它。

访问令牌

你可以在仪表板的 Access Tokens 中创建访问令牌。令牌是一串以 ts_ 为前缀的长随机字符串。

完整的令牌仅在创建时显示一次。Temporis 在服务端只存储它的哈希值，而绝不存储令牌本身。这意味着丢失的令牌无法找回——如果你弄丢了它，请将其删除并创建一个新的。

像对待密码一样对待令牌

任何持有你令牌的人都可以摄取数据并运行预测——而预测会按你的账户计费。请相应地妥善保管它们。

验证请求

在每个请求中通过 Authorization 头发送令牌，使用 Bearer 方案：

Authorization: Bearer ts_your_token_here

本文档中的示例都从 TEMPORIS_TOKEN 环境变量读取令牌，这样密钥就不会出现在源码中。在你的 shell 中导出一次即可：

export TEMPORIS_TOKEN="ts_your_token_here"

然后将其作为请求头附加上去：

curl https://api.temporis.co/v1/predict \
  -H "Authorization: Bearer $TEMPORIS_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{ "data_profile": "hourly_orders", "count": 5, "temperature": 1.0, "top_p": 0.9 }'
import os, requests

token = os.environ["TEMPORIS_TOKEN"]

resp = requests.post(
    "https://api.temporis.co/v1/predict",
    headers={"Authorization": f"Bearer {token}"},
    json={"data_profile": "hourly_orders", "count": 5, "temperature": 1.0, "top_p": 0.9},
)
resp.raise_for_status()
const token = process.env.TEMPORIS_TOKEN;

const resp = await fetch("https://api.temporis.co/v1/predict", {
  method: "POST",
  headers: {
    "Authorization": `Bearer ${token}`,
    "Content-Type": "application/json",
  },
  body: JSON.stringify({ data_profile: "hourly_orders", count: 5, temperature: 1.0, top_p: 0.9 }),
});

身份验证失败时

如果 Authorization 头缺失或格式错误，或者令牌无法识别，API 会返回 HTTP 401 以及一个描述出错原因的 JSON 主体：

{ "detail": "Invalid access token." }

缺失或格式错误的请求头则返回 { "detail": "Missing access token." }。两者都是 401。完整的响应列表请参阅错误与状态码。

妥善保管令牌

一些好习惯能让你的令牌远离不法之手：

  • 切勿将令牌提交到版本控制系统，也切勿将其放入客户端代码中。浏览器和移动应用会将其源码分发给用户。
  • 从环境变量或密钥管理器中读取令牌。让它们远离日志、截图和工单。
  • 为每个服务创建单独的令牌，这样你就可以吊销其中一个而不影响其他。
  • 在仪表板中删除令牌即可立即吊销它。删除会立即生效——下一个使用该令牌的请求将收到 401。
