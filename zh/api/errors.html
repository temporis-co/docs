错误与状态码

Temporis 如何报告成功与失败、每个状态的含义，以及如何处理它们。

响应形态

响应的形态取决于发生了什么：

  • 成功的摄取返回 204 No Content，无主体。
  • 成功的预测返回 200，带有一个 JSON 主体。
  • 任何错误都返回相应的 HTTP 状态码，并带有形如 { "detail": "..." } 的 JSON 主体。
{ "detail": "Data profile not found." }

状态码

状态含义出现位置
200 OK预测成功；主体中包含样本。predict
204 No Content摄取成功；无主体。ingest
400 Bad Request预测参数无效。predict
401 Unauthorized访问令牌缺失或无效。所有请求
402 Payment Required未绑定信用卡，或存在付款问题。predict
404 Not Found数据源未找到（ingest）；数据画像未找到或尚未就绪（predict）。ingest、predict
409 Conflict数据源已归档（ingest）；数据不足（predict）。ingest、predict
502 Bad Gateway预测失败——服务端的瞬时错误。predict

处理错误

大多数失败都对应一个明确的下一步操作：

  • 401 — 检查 Authorization 头是否存在且令牌是否有效。参阅身份验证。
  • 402 — 在仪表板的 Billing 中添加或修正信用卡，然后重试。
  • 404 — 确认数据源或数据画像存在，并且画像已就绪可用。
  • predict 上的 409 — 摄取更多历史数据，直到画像满足其最低要求，然后重试。
  • ingest 上的 409 — 数据源已归档；请取消归档，或写入另一个数据源。
  • 502 — 瞬时错误。请使用指数退避策略重试。

对状态码做一个简单的分支判断即可覆盖以上所有情况：

import time, requests

def predict(payload, token, retries=3):
    for attempt in range(retries):
        resp = requests.post(
            "https://api.temporis.co/v1/predict",
            headers={"Authorization": f"Bearer {token}"},
            json=payload,
        )
        if resp.status_code == 200:
            return resp.json()
        if resp.status_code == 502:
            time.sleep(2 ** attempt)   # transient: back off and retry
            continue
        detail = resp.json().get("detail", "")
        if resp.status_code == 402:
            raise RuntimeError(f"Billing: {detail} — add a card in the dashboard.")
        if resp.status_code == 409:
            raise RuntimeError(f"Not ready: {detail} — ingest more history.")
        resp.raise_for_status()        # 400 / 401 / 404: fix the request
    raise RuntimeError("Prediction failed after retries.")

重试

重试该重试的内容

502 可以安全重试——它是瞬时的。摄取是幂等的（写入会在 (timestamp, name) 键上执行 upsert），因此重试失败的摄取始终是安全的。请勿盲目重试 4xx 响应；应先修正请求。
