摄取记录

将观测值发送到数据源中。该数据源必须已存在（在仪表板中创建）且未被归档。

POST/v1/data_sources/ingest

请求主体

主体指定目标数据源，并携带一批记录。

data_sourcestring必填

要写入的、已存在且未归档的数据源名称。

recordsarray必填

要存储的记录列表。每条记录是某个序列在某一时刻的单个观测值。

记录字段

timestampinteger必填

以秒为单位的 Unix 时间。

namestring必填

序列名称。一个数据源可容纳多个命名序列。

valuenumber必填

观测到的数值（整数或浮点数）。

请求示例

下面的示例将四个 orders 观测值写入 store_metrics 数据源，并从环境变量读取令牌：

curl https://api.temporis.co/v1/data_sources/ingest \
  -H "Authorization: Bearer $TEMPORIS_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "data_source": "store_metrics",
    "records": [
      { "timestamp": 1718841600, "name": "orders", "value": 42 },
      { "timestamp": 1718845200, "name": "orders", "value": 51 },
      { "timestamp": 1718848800, "name": "orders", "value": 47 },
      { "timestamp": 1718852400, "name": "orders", "value": 63 }
    ]
  }'

响应

成功时，该端点返回 204 No Content 且主体为空：

204 No Content

Upsert 行为

每条记录以 (timestamp, name) 这对组合作为键。重新发送一条具有相同时间戳和序列名称的记录会覆盖已存储的值。

重试与回填是幂等的

由于写入是 upsert，你可以安全地重试失败的请求或重新发送历史数据——重复项会折叠到同一个键上，而不会累积。

批处理

相比每个请求只发送一条记录，更建议在单个请求中发送多条记录。批处理能大幅减少往返次数和开销。加载历史数据时，请按每块包含多条记录的方式分块发送，而不要一次只流式发送一个数据点。

错误

错误会返回 HTTP 状态码以及形如 { "detail": "..." } 的 JSON 主体。

状态detail 消息含义
401Invalid access token. / Missing access token.身份验证失败——请检查你的令牌。
404Data source not found.所指定的数据源不存在。请先在仪表板中创建它。
409Data source is archived.该数据源已归档，无法接受写入。请取消归档，或改用其他数据源。

完整参考请参阅错误与状态码。
