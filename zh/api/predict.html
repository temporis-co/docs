创建预测

从数据画像中抽取一条或多条采样的未来轨迹。需要已绑定信用卡。

POST/v1/predict

请求主体

你需要指定一个数据画像，以及所需样本的形态。

data_profilestring必填

你的某个画像的名称，或一个公共画像，例如 temporis/polymarket。

countinteger必填

要抽取的独立未来轨迹数量。必须至少为 1。

temperaturenumber必填

采样温度，大于 0。值越高，样本越分散；值越低，样本越接近最可能的路径。

top_pnumber必填

核采样（nucleus sampling）截断值，取值范围 0 到 1。值越低，越会裁掉不太可能的尾部；1.0 则保留完整分布。

count、temperature 和 top_p 在预测中有深入说明。

请求示例

下面的示例以中性温度从 hourly_orders 画像中抽取五条轨迹：

curl https://api.temporis.co/v1/predict \
  -H "Authorization: Bearer $TEMPORIS_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "data_profile": "hourly_orders",
    "count": 5,
    "temperature": 1.0,
    "top_p": 0.9
  }'

响应字段

请求成功时返回 HTTP 200，主体如下。

request_idstring

本次预测请求的唯一标识符。

data_profilestring

回显你所请求的画像名称。

predict_frominteger

Unix 秒：预测所从其继续的最后一个观测时间戳。每条轨迹都从这里向前推进。

predictionsarray

采样得到的轨迹。其长度等于你请求的 count。

预测对象

lossnumber

值越低，表示模型认为这条轨迹越典型。可用它来排序或挑选一个有代表性的样本。

dataarray

预测的各行。每一行包含一个 timestamp，外加画像中每个序列名称对应的一个字段。

公共数据画像

你可以无需任何自有设置，通过将公共画像名称作为 data_profile 传入来对共享数据集进行预测。这些画像由 Temporis 维护，可随时使用：

{ "data_profile": "temporis/polymarket", "count": 3, "temperature": 1.0, "top_p": 0.9 }

错误

错误会返回 HTTP 状态码以及形如 { "detail": "..." } 的 JSON 主体。

状态detail 消息含义
400Invalid prediction parameters.某个参数超出范围，例如 count 小于 1 或 temperature 不大于 0。
402Add a credit card before using the API. / There is a payment issue. Add a new credit card.没有有效的计费方式。请在仪表板中添加或修正信用卡。
404Data profile not found.不存在该名称的画像。
404Data profile is not ready to serve.画像已存在但尚未完成准备。
409Not enough data for this data profile.该画像的历史数据少于其最低要求。请摄取更多数据后再重试。
502Prediction failed.服务端的瞬时故障。可使用退避策略安全重试。

完整参考请参阅错误与状态码。
