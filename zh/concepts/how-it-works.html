预测的工作原理

Temporis 进行预测时，会把你的时间序列当作一个由大模型读取并继续延伸的序列——就像语言模型续写文本一样。本页将在你接触 API 之前帮助你建立起这种思维模型。

预测就是序列预测

语言模型读取一段文本，然后预测接下来会出现什么。Temporis 对数字做的是同样的事情。它读取你序列的近期历史，然后一步一步地继续延伸它。

该模型已经见过种类极其繁多的时间序列——周末激增的需求、随机漂移的价格、按日循环的能耗负荷。它从所有这些数据中学到了：给定之前发生的情况，接下来往往会发生什么。于是预测就归结为一个统一、通用的问题：给定这段历史，怎样的延续才是合理的？

这是一种与经典预测不同的工作方式。通常你需要先选择一个模型族，再针对单条序列逐一拟合其参数——挑选趋势项和季节项，调优它们，并在数据发生变化时重新拟合。Temporis 跳过了针对每条序列的建模过程。你把历史交给它，它直接继续延伸这个序列，借助的是从大量序列中学到的模式。

从数字到 token

在模型能够读取你的序列之前，原始数值会被转换成一串离散的 token——而模型的输出又会被转换回数字。整条流水线分阶段运行：

重采样
清洗序列并将其对齐到规则的时间网格上，使各点之间的间距保持一致。
归一化
将数值重新缩放到一个从你数据的典型分布范围（大致是第 1 到第 99 百分位区间）学习而来的统一范围，使不同的序列在模型看来具有可比性。
分块与压缩
把网格切分成一个个 patch，并用频域变换压缩每个 patch。patch 是模型每次读取和写入的单位。
量化
将压缩后的 patch 映射到一个由离散 token 组成的词表中——也就是模型实际看到的字母表。
继续延伸
模型读取 token 序列并生成其延续，就像语言模型续写文本一样。
解码
将生成的 token 转换回数字，撤销归一化，使预测落回你数据的真实尺度上。

patch 是这里的关键思想：它决定了模型读取和写入的单位。模型不是一次移动一个原始观测值——它一次移动一个 patch，这就是预测以步进而非单点方式推进的原因。

这种编码方式由你掌控

数据画像上的设置——它的 interval、patch_len 和 patch_spacing——决定了你的原始数据如何被重采样并切分成上述 patch。调优数据画像就是你为自己的序列调整编码方式的途径。

为什么是一个分布，而不是单个数字

生成就是采样。每次模型继续延伸序列时，它都会抽取 token，而不是锁定到某个固定答案，因此在同一段历史上跑两次可能产生两种不同的未来。这是一项特性，而非噪声。

一次预测请求会同时要求生成多条延续。count 参数设定要抽取多少条独立的轨迹；每一条都是一条完整且自洽的通向未来的路径。它们共同构成一个概率性预测：是一团合理结果的云，而不是单独一条线。

有两个控制项决定这些样本是如何被抽取的。temperature 设定每一步的冒险程度——越高样本越分散，越低则样本越集中在模型最可能的路径附近。top_p 在采样前裁剪掉不太可能的尾部。每条返回的轨迹还带有一个 loss，用于衡量模型认为它有多典型，从而让你能挑选一条有代表性的路径或对整组样本加权。

真正的现实价值在于这种分散范围。你看到的不再是一个几乎肯定略有偏差的猜测，而是模型认为合理的未来区间——足以让你读出最好情况、最坏情况，以及不确定性究竟有多大。

关于如何读取响应，参见预测与采样；关于如何把一组轨迹转化为区间，参见使用预测样本。

数据画像的位置

数据画像是一份已保存的配方，它把你的原始数据与上述编码方式绑定在一起。它指向一个数据源，指定要读取的序列，并固定那些驱动重采样和分块的旋钮——这样针对该数据画像的每一次预测都会以相同的方式读取你的数据。

关键的旋钮包括 interval（以秒为单位的重采样桶）、patch_len 和 patch_spacing（每个 patch 的大小和步长）、num_patches（作为上下文读取多少历史），以及针对呈乘性增长序列的 use_diffs 等变换。完整列表及每一项的选择指南，参见数据画像。
