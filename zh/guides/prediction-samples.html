使用预测样本

/v1/predict 会返回众多采样轨迹。本指南把它们转化为你真正需要的东西：一个点预测和一个置信区间。

为什么会有众多轨迹

Temporis 通过采样来生成预测 —— 它读取你的历史并描绘出一个合理的延续，就像语言模型续写文本一样。每一次抽样都是一种可能的未来，而不是那个唯一的未来。请求 count 次抽样，你就会在 predictions 中得到 count 条相互独立的轨迹。

拥有众多轨迹的意义在于其离散程度。在这些轨迹一致的地方，模型很有信心；在它们扇形展开的地方，未来确实充满不确定性。这种离散程度就是你的不确定性估计 —— 这也正是单个样本很少是你想要的原因。

把 count 设得足够高，以分辨出这种离散程度。寥寥几次抽样只能给出抖动、不可靠的画面；对大多数序列而言，20 到 100 这个范围内能很好地分辨分布。样本越多越平滑，但并非免费。

Count 会消耗 token

预测按模型 token 计费，而每条轨迹都会生成一整段预测跨度的 token。把 count 翻倍大致会使该次调用的成本翻倍。使用足够看清离散程度的样本数，然后就停下。

点预测

要把样本压缩成每个未来时间戳上的一个数字，请在每个时间步上跨轨迹做聚合。中位数是一个不错的默认值 —— 它对偶尔出现的极端轨迹很稳健；如果你更偏好均值，均值也可以。

每条预测的 data 行都以相同的顺序共享相同的时间戳，因此你可以按索引把它们对齐。这里我们从 hourly_orders 画像请求 50 个样本，并计算每个时间戳上 orders 的中位数预测：

import os, requests
from statistics import median

token = os.environ["TEMPORIS_TOKEN"]

resp = requests.post(
    "https://api.temporis.co/v1/predict",
    headers={"Authorization": f"Bearer {token}"},
    json={"data_profile": "hourly_orders", "count": 50, "temperature": 1.0, "top_p": 0.9},
)
resp.raise_for_status()
body = resp.json()

# Each prediction's rows are in the same timestamp order, so transpose by index.
preds = body["predictions"]            # length == count (50)
rows = list(zip(*[p["data"] for p in preds]))  # group row i across all samples

point_forecast = []
for step in rows:
    ts = step[0]["timestamp"]          # shared timestamp for this step
    values = [row["orders"] for row in step]
    point_forecast.append({"timestamp": ts, "orders": median(values)})

for f in point_forecast[:3]:
    print(f["timestamp"], round(f["orders"], 1))

置信区间

中位数给你一条中心线；区间给你它周围的范围。请跨样本计算每个时间戳的分位数 —— 第 10、第 50 和第 90 百分位是常见的选择。结果是三条线：一条下边界（p10）、中心线（p50）和一条上边界（p90）。

把这个区间理解为"在这个时间戳上，约 80% 的采样未来落在 p10 和 p90 之间"。区间窄意味着模型很有信心；随着你看向更远的未来而逐渐变宽的区间，是模型在诚实地告诉你：遥远的未来比近期更不确定。

import os, requests
import numpy as np

token = os.environ["TEMPORIS_TOKEN"]

resp = requests.post(
    "https://api.temporis.co/v1/predict",
    headers={"Authorization": f"Bearer {token}"},
    json={"data_profile": "hourly_orders", "count": 50, "temperature": 1.0, "top_p": 0.9},
)
resp.raise_for_status()
body = resp.json()

preds = body["predictions"]
timestamps = [row["timestamp"] for row in preds[0]["data"]]

# Shape: (count, horizon) — one row per sample, one column per timestep.
matrix = np.array([[row["orders"] for row in p["data"]] for p in preds])

p10, p50, p90 = np.percentile(matrix, [10, 50, 90], axis=0)

band = [
    {"timestamp": ts, "p10": lo, "p50": mid, "p90": hi}
    for ts, lo, mid, hi in zip(timestamps, p10, p50, p90)
]
for b in band[:3]:
    print(b["timestamp"], round(b["p10"], 1), round(b["p50"], 1), round(b["p90"], 1))

在实践中选择 count、temperature、top_p

有三个旋钮塑造样本，它们各司其职：

  • count 控制分辨率 —— 即对离散程度采样的精细程度。它不会改变分布的形状，只改变你看清它的清晰度。
  • temperature（大于 0）控制离散程度 —— 较高的值会描绘出更多样、范围更广的未来；较低的值会让样本聚集在模型最可能的路径附近。1.0 左右是中性的。
  • top_p（0 到 1）修剪尾部 —— 调低它会切掉不太可能的极端情况，从而减少令人意外的跳变；1.0 保留完整的分布。

根据你序列的跳动程度，合理的起点如下：

序列特征counttemperaturetop_p
稳定、平滑（平稳需求）20–300.80.9
典型501.00.9
波动、尖峰（价格、突发流量）80–1001.10.95

把这些当作一个起点。画一个区间，看看它是否窄到不够诚实，或宽到没有用处，然后先调整 temperature。

按 loss 排序

有时你想要的是一条单一的代表性路径，而不是一个区间 —— 用于演示、合理性检查，或快速看看"典型的未来是什么样"。每条轨迹都带有一个 loss，而 loss 越低意味着模型认为该路径越典型。按它排序并取列表前部：

# body is the parsed /v1/predict response
ranked = sorted(body["predictions"], key=lambda p: p["loss"])

most_typical = ranked[0]               # lowest loss = most representative path
for row in most_typical["data"][:3]:
    print(row["timestamp"], row["orders"])

# Or trim outliers: keep the most-typical 80% before computing a band.
keep = ranked[: int(len(ranked) * 0.8)]

对于一个真正的预测，相比任何单条轨迹，更应优先选择中位数和区间 —— loss 最低的路径只是一种合理的未来，而不是它们的平均。排序最有用的场景是挑选一个干净的示例或过滤离群值。
