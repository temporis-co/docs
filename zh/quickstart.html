快速上手

本指南带你从一个全新的账户走到你的第一份预测。你将创建 一个 API 密钥、注册一个数据源、流式发送一些历史数据、构建一个数据画像， 并调用预测端点。预计需要大约五分钟。

两个界面，一个平台。你将在控制台中完成各项设置（密钥、数据源、数据画像、计费），并通过 API 完成数据与预测工作。本页会带你走完这两部分。

  1. 创建一个访问令牌

    在控制台中打开 Access Tokens 并创建一个。该令牌 只会显示一次——请立即复制。它形如 ts_…， 用于对每一个 API 请求进行身份验证。

    将它存为环境变量，以便下面的示例可以直接运行：

    export TEMPORIS_TOKEN="ts_your_token_here"

  2. 添加付款方式

    预测按使用的每个 token 计费，因此在 /v1/predict 运行之前需要先绑定一张银行卡。请在 Billing 下添加。 导入数据不需要计费——只有预测才需要。

  3. 创建一个数据源

    在 Data Sources 下，创建一个名为 store_metrics 的数据源。 名称必须以小写字母开头，且只能包含小写字母、 数字和下划线。在你向其发送数据之前，数据源只是一个空容器。

  4. 发送一些历史数据

    每条记录都是一个 (timestamp, name, value) 三元组。 timestamp 是以秒为单位的 Unix 时间，name 是序列名，value 是数值。这里我们发送少量 每小时的 orders 读数；实际使用时，你应当尽可能多地发送真实 历史数据。 

    curl -X POST "https://api.temporis.co/v1/data_sources/ingest" \
  -H "Authorization: Bearer $TEMPORIS_TOKEN" \

  5. 构建一个数据画像

    回到控制台，打开 Data Profiles 并创建一个名为 hourly_orders 的画像。把它指向 store_metrics 数据源， 选择 orders 序列，并将 interval（时间间隔） 设为 3600 秒（一小时）。其余各项的默认值 作为起点都很合适。

    在 Temporis 能够将画像准备好以供服务之前，它需要一段历史作为基线。 一旦数据足够，控制台会将其标记为 ready（就绪）。在此之前， 预测会返回 "not ready" 错误——这是预期之中的。

  6. 生成一份预测

    现在向画像询问接下来会发生什么。count 表示要抽取多少条独立的 未来轨迹；temperature 和 top_p 控制 采样的大胆程度。

    "temperature": 1.0,
    "top_p": 0.9

刚刚发生了什么

你将观测值流式写入了一个数据源，用一个数据画像描述了如何读取 它们，并从 /v1/predict 抽取了五种可能的未来。每一条预测 都是一条完整的轨迹；它们合在一起构成了对接下来走势的 概率性图景。一条预测的 loss 越低，模型就认为这条轨迹 越典型。
